# FIBR-0142 — Recurring money detection

Closed 2026-07-15 by `/close-phase`. Spec: `docs/specs/FIBR-0142.md`
(cold-eyes converged loop 5). Split from FIBR-0022 (budgets remain the
FIBR-0022 follow-up).

## What changed

Auto-detects repeating money movements — recurring OUT (subscriptions,
debit orders, insurance) and recurring IN (salary, standing deposits) —
and surfaces them for the user to Confirm or Dismiss.

- **Pure detector** (`services/recurring.py`): `detect_recurring(rows,
  today, exponent, excluded_ids)` groups by `(direction,
  merchant_key)` where `merchant_key =
  normalise_text(merchant_name(description))` (reuses FIBR-0138),
  qualifies under the Balanced rule (≥3 members; every magnitude within
  ±10% of the integer `median_low`, an integer cross-multiply — no
  float; ≥2 non-zero day-gaps all in one cadence band
  weekly/fortnightly/monthly/yearly), filters to active arrangements,
  and returns items sorted biggest-first. Clock-injected (`today` is a
  param), no I/O, deterministic. Monthly-equivalent per D8
  (`amount × N / 12`, `ROUND_HALF_EVEN`).
- **Schema v9** (`_migrate_to_v9`): the `recurring_decisions` table,
  keyed on `(direction, merchant_key)` (not txn ids) so a decision
  survives re-imports; no FK/cascade. One atomic step; drift-guard
  assertions across every feature bumped 8→9.
- **`RecurringRepository`**: the detector's full-history/all-accounts
  feed (`recurring_rows`) + the decision store (upsert / clear). Houses
  `_RecurRow` (re-exported from the service) to break the
  service↔repository import cycle.
- **`RecurringService`**: wraps the detector with the vault exponent +
  the confirmed-transfer exclusion set, and partitions its output in
  one `snapshot(today)` pass into Suggested / Confirmed / (hidden)
  Dismissed; `confirm`/`dismiss`/`reset`; `summary` (per-month totals
  for the deferred FIBR-0143 card; `net = in − out`).
- **Recurring tab** (`ui/recurring.py`): a `RecurringWidget` mirroring
  the Transfers tab — Suggested (Confirm/Dismiss) over Confirmed
  (Un-confirm); columns Merchant / Direction / Cadence / Amount / Next
  due / Seen; direction & cadence render `tr()`-ed labels off stable
  ASCII tokens; every slot catches `VaultLockedError`; empty-state
  message. Wired after Transfers in `main_window.py` (8-tab workspace,
  new `recurring.svg` toolbar glyph).

TDD: `tests/features/recurring/` — pure detector red→green first, then
migration → repository → service → tab.

## What was learned

- **A pre-divided `Decimal` factor is a rounding hazard.** Storing
  `Decimal(26)/Decimal(12)` rounds the *factor* at the 28-sig-fig
  context, which then defeats `ROUND_HALF_EVEN` for exact-half
  fortnightly amounts (~2.9% off by one minor unit). The fix — and the
  spec's D8 form — keeps the division in-line (`amount × N / 12`) so
  `amount × N` stays exact. The original half-even test used a *yearly*
  case, which has zero divergence and passed either way; the cold review
  caught the untested fortnightly path.
- Every schema bump ripples into ~24 per-feature drift-guard assertions
  (and their version-encoded test names). This is an established but
  fragile convention — see "What was deferred".

## What was deferred

- **Home dashboard card → FIBR-0143** (the dashboard-focus rework). The
  `RecurringSummary` dataclass and `RecurringService.summary()` a card
  consumes are built and tested here, so FIBR-0143's card is a thin
  consumer.
- **Budgets → FIBR-0022** (the other half of the original split).
- **Schema drift-guard fragility** (noted, not filed): each schema bump
  forces edits to a `LATEST_SCHEMA_VERSION == N` assertion in every
  feature that ever added a migration. Centralising this to one test
  would remove the per-bump churn — a candidate cleanup, out of scope
  for this feature.

## Findings folded (cold code-review, at close)

- **HIGH (money):** fortnightly `_monthly_equivalent` pre-divided factor
  → wrong `ROUND_HALF_EVEN` result. Fixed (deferred division) +
  reproduce-first test.
- **LOW (contract):** `set_decision` upsert reset `created_at` on a
  status flip; spec says it's immutable provenance. Fixed + deterministic
  regression test.
- **Test strength:** a ≥2-item post-sort widget test (guards the
  tag-based selection mapping), a rendered direction/cadence label
  assertion, a stronger `tr()`-wrap grep.
