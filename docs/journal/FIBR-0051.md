# FIBR-0051 — P07.5 app-shell UX redesign (journal)

Closed 2026-07-09 by `/close-phase`. Tag `FIBR-0051-complete`.

The first of a 4-piece body of work (shell → P08 rules → P09 transfers → P10
dashboard); it ships **first** so the later features drop into the finished
shell with no rewiring.

## What changed

- **`ui/main_window.py` rewritten** from `AppShell(QStackedWidget)` (a
  full-screen panel-swapper) into **`MainWindow(QMainWindow)`** — a real desktop
  window: menubar (File · View · Help · Donate), an icon toolbar
  (`ToolButtonTextUnderIcon`), a central `QStackedWidget` content area, and a
  status bar. It owns the startup state machine
  (`_show_first_run`/`_show_unlock`/`_enter_unlocked`/`_lock`/`_show_home` +
  `_set_live`/`_clear_live`), the single tracked-dialog slot (`self._dialog`),
  the Donate URL constants, and the status-bar count widget.
- **First-run + unlock re-homed** `QWidget → QDialog` (`FirstRunDialog`,
  `UnlockDialog`) shown non-blocking (`setModal(True)` + `show()`) over the
  window, keeping their `DeriveWorker` flow + signals verbatim. **Manual entry**
  became `ManualEntryDialog`. **`HomeView`** (`ui/home.py`) holds the
  getting-started page + the transaction table, toggled by transaction count.
- **Security spine preserved (INV-3/4/5/8):** the content widget is *destroyed*
  on lock/auto-lock (not hidden), so no `HomeView` with decrypted rows survives a
  lock; any open dialog is closed before the vault shuts; the key is still wiped
  on quit/lock/auto-lock; the Donate menu's `QDesktopServices.openUrl` is the
  sole (user-initiated) egress — the app opens no socket.
- **Icons:** hand-authored monochrome SVG toolbar glyphs (`ui/icons/*.svg`) +
  loader (`ui/icons.py`), shipped as package data (wheel + PyInstaller
  `--add-data`); `_selftest.py` gained an icon-render leg proving they travel.
- **Tests:** `tests/features/app_shell/` (22 tests, INV-1..10 + sub-invariants);
  the D10 ripple re-homed `test_vault.py` + `test_accounts.py` assertions to the
  split widgets (mechanical, no coverage lost) plus the genuine rewrite of the
  auto-lock routing test to INV-4a's outcome.
- **Roadmap reconciliation:** closes **FIBR-0040** (donate links delivered by the
  Donate menu); **FIBR-0049** stays open, annotated (its getting-started welcome
  shipped here; the inline Amount/Description field hints remain).

## What was learned

- **A transient `QStatusBar.showMessage(text, timeout)` clears to *empty*, not to
  a resting message.** The first draft's "Ready" went blank after the first
  transient. Fixed by restoring "Ready" off `QStatusBar.messageChanged` when the
  message area empties (INV-7). Caught by indie-review lane A.
- **`QIcon(path).isNull()` is `True` for a *missing* file** (verified empirically
  in PySide6 6.11.1) — so the original rationale ("a bare `QIcon` is non-null even
  when absent") was wrong. The rendered-pixmap self-test is still the right check,
  but for a different reason: a *present* SVG whose `qsvg`/`qsvgicon` plugins were
  dropped yields a non-null `QIcon` but a **null** pixmap. Corrected the comment +
  the two matching spec parentheticals (a §13 factual doc-fix; INV/contract
  unchanged, so no cold-eyes re-loop).
- **`_format_amount` renders via the default `QLocale`,** so a naive `"12.34" in
  rendered` assertion is locale-coupled. Pinned `QLocale.c()` in the test for
  hermeticity (indie-review lane B).

## What was deferred

- **INV-2f app-quit edge (accepted, not a code change).** The dialogs neutralise
  their own dismissal routes mid-derivation, but a *main-window* teardown /
  app-quit while a derivation is in flight could still destroy the parented
  `DeriveWorker` (pre-existing FIBR-0004 behaviour). It is UI-unreachable in
  practice — the first-run/unlock dialogs are application-modal, so the main
  window's `[X]` and `File → Quit` are blocked while a worker runs. Left as-is;
  the spec's "no teardown races a live worker" scopes to reachable UI paths.
- **FIBR-0049 inline field hints** (Amount/Description) — stays open on the
  roadmap, annotated.
- Method-naming drift (`_show_content` in the spec's Deliverable 2 vs
  `_set_live`/`_open_*` in code) — INFO, no contract impact (tests key off
  `objectName`/behaviour).
