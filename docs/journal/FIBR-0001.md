## What changed

- **`pyproject.toml`** — PEP 621 metadata (`finbreak`, Python ≥ 3.12,
  SPDX `license = "MIT"`), a PEP 735 `[dependency-groups] dev` block
  exact-pinned (`ruff`, `bandit`, `pip-audit`, `pytest`, `pytest-qt`),
  and the ruff / pytest / bandit tool config. src-layout resolved via
  `pythonpath = ["src"]` — no editable install needed for the gate.
- **`scripts/ci-local.sh`** — the one-command quality + security gate.
  Stages run cheapest-first (`ruff check` → `ruff format --check` →
  `bandit` → `pip-audit` → `gitleaks dir` → `pytest -m "not perf"`),
  `set -euo pipefail` so it exits non-zero on the first failing stage.
- **`.github/workflows/ci.yml`** — CI mirror on `ubuntu-24.04`. Installs
  the dev group + the gitleaks binary, then invokes the script rather
  than re-listing stages (INV-2: single source of truth).
- **`.gitleaks.toml`** — extends the default ruleset; allowlists only
  *paths* (`.venv/`, `.git/`, `dist/`, `build/`) — narrows WHERE we scan,
  never WHAT counts as a secret.
- **`src/finbreak/__init__.py`** — placeholder package exposing
  `__version__`. **`tests/test_smoke.py`** — asserts the package imports.
- **`CLAUDE.md`** — filled the Build-and-test and Module-map sections.

## What was learned

- **The gate works as designed on day one.** `pip-audit` caught
  `pytest==9.0.2` (CVE-2025-71176); bumped the pin to `9.0.3` — a
  root-cause fix, not a suppression. INV-3 anticipates exactly this.
- **`pytest-qt` aborts collection with no Qt binding installed.** PySide6
  arrives at P02, so the plugin is pinned-but-unused now. Disabled via
  `addopts = "-p no:pytest-qt"` (the entry-point name is `pytest-qt`, not
  `pytestqt` / `qt`). A comment marks the line for removal at P02 when the
  first `qtbot` GUI test lands — honours INV-4 (pin now, use at P02)
  without letting the pin rot silently.
- **Action versions were verified, not guessed** — `checkout@v7`,
  `setup-python@v6` are current (v7.0.0 / v6.3.0), not the v5-era muscle
  memory.
