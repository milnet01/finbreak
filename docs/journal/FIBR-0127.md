# FIBR-0127 — App-wide theme system (six finance themes + follow-system) & modern polish

**Closed 2026-07-14 by `/close-phase`.** Spec `docs/specs/FIBR-0127.md` (cold-eyes
converged loop 6); ADR-0010. Tag `FIBR-0127-complete`.

## What changed

- **`src/finbreak/ui/theme.py` (new).** A theme is eight semantic colour **tokens**
  + an `is_dark` flag (`ThemeTokens`), registered per id in `THEMES` (six ids:
  Ledger · Parchment · Mint / Midnight · Graphite · Emerald). The tokens expand,
  with no per-theme hardcoding, into:
  - `build_palette(tokens)` — a full `QPalette` role map (INV-4), with a **computed**
    `HighlightedText` (near-white on a dark accent, near-black on a light one) so
    selected-row text always contrasts the accent.
  - `build_stylesheet(tokens)` — the polish QSS (INV-5/INV-11): gradient/glow accent
    borders on inputs/buttons/tabs, and hover + selection row gradients with
    alternating stripes, each gradient emitting its base token hex verbatim as one
    stop. Class-scoped so the chart `QGraphicsView` is never wrapped.
  - `polish_item_views(root)` — `setAlternatingRowColors(True)` on every
    `QAbstractItemView` descendant (turns the QSS stripes on).
  - `ThemeController(QObject)` — owns the mode (a theme id or `"system"`), applies
    Fusion + palette + stylesheet then emits `themeChanged`, and while in `"system"`
    mode follows the OS light/dark scheme **live** via `styleHints()
    .colorSchemeChanged` (Qt 6.5+). `Unknown` scheme → dark default (D5); an unknown
    id normalises the **mode** to `"system"` (D3). Parented to the app for lifetime.
  - `load_theme_pref` / `save_theme_pref` — the mode stored in the **non-vault**
    window INI (`paths.window_settings_path()`), guarded (absent/unknown →
    `"system"`), so it applies to the first, still-locked window (D2).
- **`app.py`** — `run()` now reuses a live `QApplication` (re-callable under the
  pytest-qt app) and builds the `ThemeController` + applies the stored theme
  **before** `MainWindow`, so the locked first window is themed (INV-1).
- **`ui/main_window.py`** — optional `theme_controller` keyword (default `None`, D10);
  `self._icon_actions` map (populated by `_make_action` for icon-bearing actions);
  `_retint_toolbar_icons` on `themeChanged` (delivers FIBR-0116's live re-tint);
  `polish_item_views(self._workspace)` — all gated on a controller so the
  controller-less default path (every existing construction) is byte-for-byte
  unchanged. `_open_settings` passes the controller through.
- **`ui/settings.py`** — an immediate-apply theme picker (`settings_theme`): "Follow
  system" + the six themes grouped Light-then-Dark, preselected to the pref; its
  `currentIndexChanged`→`set_theme` connection is made **only after** populate +
  preselect, so opening Settings can't clobber a pinned pref (INV-8, D4).
- **D11 citation fixes** — the four stale ADR-0002 dark-default comment cites
  (`ui/icons.py`, `ui/home.py`, `ui/_amount.py`, `services/pdf_export.py`)
  re-pointed to **ADR-0010** (the real "the app is dark" decision). Comment-only.
- **`docs/decisions/0010-theme-system.md`** — Accepted; README index row.
- **`tests/features/theme/`** — spec.md + 30 conformance legs (INV-1..13 + D3/D4)
  under a `theme_isolation` palette/style/stylesheet restore fixture (D12).

## What was learned

- **A Qt stylesheet masks the applied base style.** Once `set_theme` installs the
  app-level QSS, `QApplication.style()` is Qt's internal `QStyleSheetStyle` proxy
  whose `objectName()` **and** `name()` both return `""` — the Fusion base is not
  introspectable through the proxy (verified in-venv, Qt 6.11.1). The spec's INV-1 /
  INV-6 originally asserted Fusion via `style().objectName().lower()=="fusion"`,
  which cannot pass. Corrected (never-silently-drift): INV-6 pins Fusion via a spy
  wrapping `QApplication.setStyle`; INV-1's recorder uses the pinned-theme palette
  as the before/after-window ordering falsifier. No design change.
- **`QSettings` round-trips within a process without an explicit `sync()`** (shared
  in-process cache), so `save_theme_pref` can honour D4 (no `sync()`/`status()`
  inspection) and INV-2's save→load round-trip is still deterministic.
- **The offscreen headless default palette is light** (`Window` L≈0.94) with a
  `fusion` base style — so the `theme_isolation` snapshot-at-setup restores to a
  known clean state, and the ledger(light)→midnight(dark) flip is a genuine change.

## What was deferred

- **Per-theme amount re-tinting** (`_amount.py` / `home.py` red/green from theme
  tokens) — the FIBR-0116-adjacent follow-up; the fixed mid-tones stay (D9).
- **A literal *blurred* glow** (`QGraphicsDropShadowEffect` on discrete widgets) —
  Qt has no CSS `box-shadow`; the gradient/glow is delivered with accent gradients
  (D6).
- **Hyperlink legibility** — a cold-review INFO: `Link` ← `accent_soft` (INV-4) is
  the pale companion token, so rendered hyperlinks are low-contrast against `base`.
  Spec-conformant (no styled links exist in the vault UI today), noted for a design
  second-look if links are ever surfaced.
