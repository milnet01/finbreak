# FIBR-0058 — ofxparse findAll DeprecationWarnings — closed 2026-07-09

**Kind:** chore (dependency hygiene). ~107 `DeprecationWarning: Call to
deprecated method findAll` per test run, from ofxparse calling BeautifulSoup's
long-deprecated `findAll` (ofxparse.py:445/449/454/949).

## Investigation

- **ofxparse 0.21 is the latest PyPI release** (checked: `pip index versions
  ofxparse` → 0.21 is LATEST). Effectively unmaintained (0.21 dates to 2023) —
  no upstream `find_all` fix to track (roadmap option (a) is a dead end).
- **beautifulsoup4 4.15 still ships `findAll`** as a deprecated alias — it works,
  it only warns. The deprecation is "since 4.0.0"; removal is slated for **bs4
  5.0**. bs4 is an **unpinned transitive dep** (via ofxparse), so a future
  install could pull bs4 5.x and **break OFX import** — the real latent breakage,
  distinct from today's cosmetic noise.
- The gate does not `-W error`, so today the warnings are noise, not failures.

## What changed (`pyproject.toml`, config-only)

- **Scoped `filterwarnings`** (roadmap option (b)) — `[tool.pytest.ini_options]`
  ignores **only** `"Call to deprecated method findAll.*"` (a specific message,
  not a blanket `DeprecationWarning` silence), so our own future deprecations
  still surface. 107 → 0 warnings.
- **`beautifulsoup4>=4.9,<5` pin** — the latent-breakage guard. Caps bs4 below
  the 5.0 that removes `findAll`. bs4 is only reached transitively through
  ofxparse (finbreak never imports it directly), so this is a compatibility
  constraint, documented as such.

## What was learned

- The warning is two problems wearing one coat: **noise** (fix by scoping the
  filter) and **latent breakage** (fix by capping the transitive dep). Only
  quieting the warning would have left the real risk — a future `pip install`
  pulling bs4 5.0 — in place. Naming both and addressing each at its root beat a
  one-line `# ignore`.

## What was deferred

- **Replacing ofxparse** (roadmap option (c)) — large: FIBR-0008 built the OFX
  import pipeline around it. Revisit only if ofxparse stays unmaintained *and*
  bs4 5.0 becomes necessary for another reason. Noted, not scheduled.
