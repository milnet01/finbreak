# FIBR-0146 — Statement date-format auto-detect + confirm

Closed 2026-07-17 by `/close-phase`. Kind: fix. Spec:
[`docs/specs/FIBR-0146.md`](../specs/FIBR-0146.md) (cleared for code after
`/cold-eyes` 7 loops × 3 lanes).

## What changed

The tester's bug: the import wizard asked for the date layout as a raw
programmer code (`%Y-%m-%d`), so a bank printing dates day-first failed **every**
row with the raw Python message `time data '…' does not match format '…'`
(0 new · 0 duplicate · 165 error). The pipeline downstream of the date read was
correct — only the guess and the failure message were wrong.

Shipped by TDD in five slices:

1. **`importers/date_detect.py`** (new, pure — no Qt/vault) — `detect_date_format(Sequence[str]) -> DateFormatGuess`
   over 15 ordered `KNOWN_DATE_FORMATS`. Format-outer scan, highest parse-count
   wins, ties by fixed list order (deterministic), `None` if nothing parses.
   **No year-plausibility guard** — `%Y`/`%y` match exactly 4/2 digits, so
   `strptime` itself separates the widths (verified against CPython). Clock-free
   (an INV-2 falsifier test poisons `date.today()`).
2. **`importers/csv_importer.py`** — the date parse split into its own `try`, so a
   failure surfaces a friendly `RowError` (`could not read the date "<raw>"`, or
   `the date cell is empty`) with no `%`-format internals; amount/`parse_transaction`
   messages unchanged.
3. **`ui/import_wizard.py`** — the raw `QLineEdit` replaced by a plain-English
   `QComboBox` of example dates (data = fixed `%` pattern) + a **"Custom…"** reveal
   for exotic banks; auto-detect on entry (unmatched CSV / generic PDF) and on a
   manual date-column change; a live **"Dates read as: …"** preview with two
   fallbacks + an ambiguity nudge; a whole-import **banner** when nothing lands.
   Single-owner signal wiring; matched profiles stay authoritative.
4. **`services/import_.py`** — `_validate_mapping` rejects an empty/blank
   `date_format` (`"choose a date format"`), closing the `strptime("", "")` →
   1900-01-01 phantom-date trap.
5. **`tests/features/import_date_detect/`** — spec.md + 39 tests across the four
   layers (detector / importer / service / wizard-qtbot).

## What was learned

- **`strptime` separates 2- and 4-digit years by field width, not by value** —
  the cold-eyes loop had killed a plausible-year guard built on a false claim;
  TDD confirmed the guard was genuinely unnecessary (the width-separation tests
  are green with zero guard code). Simplest correct implementation held.
- **`isVisible()` is the wrong probe for an unshown page.** A widget on a
  `QStackedLayout` page that is never the current page reads `isVisible() == False`
  even after `setVisible(True)`. The tests use `isHidden()` (the explicit-hide
  flag), which reflects the `setVisible` intent regardless of ancestor state.
- **A test can silently mask the bug it should catch.** The first
  `test_matched_profile_is_authoritative` put the date at column 0, so the
  programmatic combo set was a no-op and never exercised the unblocked slot — the
  MEDIUM wiring bug (below) passed the test. Moving the date off column 0 + spying
  on auto-detect turned it into a real falsifier.

## Close review (2 cold lanes)

- **Lane A (date-parsing / money / edge-cases): no reachable defects.** The tie
  logic, the width separation, the friendly-error isolation, and the 1900 trap
  closure were all verified sound against CPython + the code.
- **Lane B (Qt wiring / test quality): 2 MEDIUM + 2 LOW, all folded inline:**
  - MEDIUM — `_apply_profile_to_combos` set the date combo **unblocked**, re-firing
    `_on_date_column_changed` (spurious re-detect + double preview refresh) on a
    matched profile whose date column isn't column 0 — contradicting spec D5/INV-4.
    The final state was corrected two lines later, so it was **never a wrong-day**,
    but the wiring contract was broken. Fixed with a `QSignalBlocker`.
  - MEDIUM — the D5(b) PDF fire points had zero tests; added
    `test_pdf_table_switch_redetects_D5b` + `test_pdf_matched_table_uses_profile_not_autodetect`.
  - LOW — the masked matched-profile test (strengthened, see above).
  - LOW — added the INV-1 "0 rows before the Import press" falsifier.

## What was deferred

- **Lane A's optional defense-in-depth suggestion** — adding a redundant empty-format
  guard *inside* `CsvImporter.parse` (in addition to the `_validate_mapping`
  boundary). Not taken: the spec (D4) deliberately placed the guard at the service
  boundary, the reviewer confirmed **no reachable bypass** in the shipped code, and
  an unreachable error path in a pure importer violates the shortest-correct rule.
  Noted here rather than added.
- The spec's own out-of-scope list stands: English month names only (i18n →
  FIBR-0017), no per-row mixed formats, no amount/debit-credit column detection,
  2-digit-year layouts slash-only in the known list (dash/dot via Custom…).
