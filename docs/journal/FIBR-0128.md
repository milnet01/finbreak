# FIBR-0128 — Forget remembered statement passwords (Accounts screen)

**Closed 2026-07-14 by `/close-phase`.** Spec `docs/specs/FIBR-0128.md` (cold-eyes
converged loop 3). No ADR — extends the FIBR-0009 store. Tag `FIBR-0128-complete`.

## What changed

- **`repositories/accounts.py`** — `ids_with_pdf_password() -> set[int]`
  (`SELECT id FROM accounts WHERE statement_pdf_password IS NOT NULL`): the presence
  query selects **only the id column**, so a listing never pulls the encrypted secret
  into memory (INV-1).
- **`services/accounts.py`** — `account_ids_with_pdf_password()` wraps it (the UI
  talks to the service, never the repo). Clearing **reuses** the existing
  `set_pdf_password(id, None)` — no new write method (§3 reuse).
- **`ui/accounts.py`** — a **Forget statement password** button on the Accounts
  screen (disabled unless the *selected* account has a saved password), a
  screen-reader-legible presence marker (`"  ·  🔑 statement password saved"`)
  appended to marked rows in `_refresh`, a new `_ACCOUNT_HAS_PW_ROLE` bool stashed
  per row, and `_on_forget_password` (defensive null-guard + `QMessageBox.question`
  confirm + clear under a `VaultLockedError` guard + `_refresh`). The enable/disable
  is recomputed in `_on_selection_changed` **before** its `None` early-return, so a
  post-Forget `_refresh` (which clears the list) disables the button rather than
  leaving it stale-enabled.
- **Tests** — 8 conformance legs in `tests/features/accounts/test_accounts.py`
  (INV-1..5) + the FIBR-0128 section of `tests/features/accounts/spec.md`.

## What was learned

- **`_on_selection_changed` early-returns on `None`** — the cold-eyes loop-1 HIGH.
  The existing handler does `if item is None: return` before its body, so button-state
  logic placed naively *after* it would never fire on a cleared selection. Fixed by
  recomputing enable/disable *before* the early return; INV-3's post-Forget leg pins it.
- **The never-show design makes the presence query the whole security story.** By
  selecting only `id` (never `statement_pdf_password`) the plaintext is kept out of
  the UI *by construction* — the INV-1 test spies that `get_pdf_password` is never
  called during render, a stronger falsifier than string-absence alone.
- **`IS NOT NULL` is safe against an empty-string false positive** — the sole capture
  site (`import_wizard.py`) guards `if remember and password:`, so a stored password
  is always truthy (confirmed during close-review).
