## What was learned

- **The spec's pinned fixture header couldn't feed the pipeline.** The draft
  header `Date, Money Out, Money In` has no description column, but the reused CSV
  pipeline rejects a blank description (`parse_transaction`), so no valid draft is
  possible — INV-1/INV-7e would be untestable. Regenerated the three fixtures with
  a `Description` column (3 data rows kept, so the cap tests still work); surfaced
  the correction in the spec's Deliverable 8 and the test spec (§14, not silent).
- **`pdfplumber` extracts owner-only PDFs directly** (the earlier "can't extract"
  reading was a probe artifact — an un-ruled table); routing through `pikepdf` is
  for the clean `PasswordError` signal + qpdf robustness, not correctness (D3).
- **The `.venv` predated the `Fin_Break`→`finbreak` dir rename**, so its
  console-script shebangs pointed at a gone directory and `ci-local.sh` failed
  with "bad interpreter". Repointed the paths (root-cause fix; a `pip install
  --group dev` regen would do the same).

## What was deferred

- Free-text (non-table) PDFs, scanned/OCR PDFs, header-less continuation
  stitching, per-bank table-detection tuning, and a background extraction thread
  — all named in the spec's § Out of scope (YAGNI; the table path covers the
  common case).
- The indie-review's informational notes (the `INV-11` no-log test is a valid
  forward guard though near-vacuous today; the caps spec-lock test; the
  stale-stored-password-on-override UX being a conscious INV-4 choice) — accepted
  as-is, no change warranted. No `docs/known-issues.md` entries.
