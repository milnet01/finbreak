# FIBR-0004 — P02 security spine (journal)

Closed 2026-07-02. The first code enforcement of the security model:
master password → Argon2id → SQLCipher vault → one manual transaction
→ table → lock.

## What changed

- **Crypto & storage.** `crypto.py` (Argon2id, pinned parameters, directional
  strength floor, sidecar validation incl. an unknown-`format_version` refusal),
  `vault.py` (one owned SQLCipher connection, vault created owner-only via
  `O_CREAT|O_EXCL, 0o600` *before* any ciphertext is written, atomic owner-only
  sidecar, wrong-key/tamper detection on first read, vault-before-sidecar
  ordering), `models.py`, `errors.py`, `paths.py`.
- **Logic.** `services/auth.py` (password↔key lifecycle; key wiped on lock /
  idle / exit; `on_auto_lock` callback so the UI can route away on an idle lock),
  `services/transactions.py` (exact minor-unit money via `Decimal`, trailing
  zeros normalized so `12.340` is accepted), `repositories/transactions.py`
  (manual-commit atomicity seam).
- **UI (PySide6).** first-run / unlock / main-window screens; key derivation on a
  `QThread` worker (parented, cleaned up, re-entrancy-guarded); amounts via
  `QLocale.toCurrencyString` with the base symbol; idle auto-lock routes back to
  the unlock screen. `app.py` routes by vault presence state.
- **Entry point.** no-args now launches the GUI (retired the FIBR-0003
  `NOT_BUILT` placeholder); the self-test grew a fourth (Argon2) native leg.
- **Tooling.** `mypy` added to the dev group with a `[tool.mypy]` config
  (per-module `ignore_missing_imports` for the stub-less native deps); `.gitignore`
  ignores `.audit_cache/`.

## What was learned

- **mypy earns its keep immediately.** Making it runnable locally (not just
  server-side in the audit) caught a real bug the audit's config missed: the
  `service` fixture yielded but was typed `-> AuthService`.
- **The perms race is at *create*, not *chmod*.** Creating the file
  `O_EXCL, 0o600` before SQLCipher writes closes both the world-readable window
  and the error path a late `chmod` left open.
- **Idle auto-lock is a two-part contract.** The service-side wipe (speced) is
  only half — without a callback to route the UI away, the assembled app crashed
  on the next click after a 10-minute idle lock. Round-1 review missed it;
  round-2 caught it. Cross-layer bugs need the app assembled, not just units.
- **Three cold review rounds converged** (HIGH → MEDIUM/defensive → doc/test),
  which is the expected shape; the tail rounds are hardening, not bug-hunting.

## What was deferred

- **Live language switching (`retranslateUi` / `changeEvent`)** → **FIBR-0017**
  (the i18n phase, annotated there). coding.md §5.2 asks for it "from P02", but
  the FIBR-0004 spec deliverable required only `tr()` strings + RTL layouts +
  `QLocale` amounts (all shipped) and there are no translations to switch yet.
  Deferred per user decision 2026-07-02.
- **User-configurable auto-lock timeout** stays a fixed `AUTO_LOCK_MINUTES = 10`
  → **FIBR-0014** (already speced as out-of-scope here).

## False positives confirmed

- None. `docs/audit-allowlist.md` remains empty — every review finding across
  three rounds was either a real fix (applied inline) or a spec-accepted
  best-effort gap (D5 memory-wiping of immutable `bytes`/`str`), now documented
  in-code rather than suppressed. One reviewer flagged the `mypy==2.1.0` pin as
  "anachronistic"; verified real (`pip index versions mypy` → 2.1.0 is latest),
  so not a finding.
