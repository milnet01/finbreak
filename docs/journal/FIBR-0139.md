## What was learned

- **`normalise_text` strips trailing whitespace**, so a `"bp "` / `"rain "` word-boundary
  trick collapses to a risky short token ‚Äî caught during implementation and those
  patterns were dropped from the seed. Substring-only matching (D2) genuinely can't
  word-bound.
- The `library_enabled` **module function** and the `CategorizationService.library_enabled`
  **method** share a name deliberately: inside the method body a bare `library_enabled`
  resolves to the module global (methods aren't in scope as bare names), so the method
  cleanly delegates to the helper.

## What was deferred

- **FIBR-0140** (learn-from-your-own-history ‚Äî statistical, no hand-written rule) remains
  the approved "later" half; this phase ships the static library only.
- Structural (id-based) binding of library entries, immediate re-file on toggle,
  regex/wildcard entries, and a user-editable/importable library are all out of scope
  (spec ¬ß Out of scope).

## Reviewed and accepted (not folded)

- **Substring false-positive guesses** (e.g. `"spar"` could match "auto spares"): the
  explicitly-accepted consequence of the D2 substring-only design (user decision
  2026-07-09, carried over). Every guess is **marked "~ guess" and overridable**, runs
  only on auto rows, and **never touches money** (INV-1 holds ‚Äî verified by the
  end-to-end conservation test). Per-merchant precision is deliberately **not** CI-checked
  (Deliverable 1); the library is a per-release data file, so a future data-only update
  can tune/drop entries with **zero code change**. Not a correctness defect ‚Äî no fix-pass.
- **The Windows freeze driver's `--add-data` emission is not directly asserted** (only the
  Linux freeze is scraped and set-checked against the flag constants). This is a
  **pre-existing structural limitation symmetric with the icons pair** (the driver builds
  its flags from the shared `windows_freeze_flags` constants), not introduced by this
  change; INV-8 makes a bundling miss non-fatal (empty library, no crash). Noted for a
  future parity-guard improvement, not folded here (¬ß11 stay-in-your-lane).
