# FIBR-0143 — Dashboard-focus rework (breakdown is the hero)

Closed 2026-07-16 by `/close-phase`. Spec: `docs/specs/FIBR-0143.md`
(cold-eyes converged, 7 loops × 3 cold lanes). Kind: ux.

## What changed

- **`ui/charts.py` — `build_breakdown_donut` (new).** A dashboard column's ≤8-wedge
  donut from generic `(label, amount)` slices, with its **own** cap-and-collapse loop
  (≤ 8 → all kept, no Other; > 8 → top 7 + one Other wedge summing the tail),
  palette-coloured with **no** reserved Uncategorised neutral, empty-safe. Deliberately
  **not** an extraction from `_donut_wedges` — that keeps the spending donut
  (`_donut_wedges` / `build_donut_chart`) byte-for-byte unchanged, so the PDF export is
  provably untouched (D3/D10).
- **`ui/home.py` — the dashboard reworked.** `_build_dashboard` now lays out selectors →
  slim **Net** strip → three side-by-side columns (Expenditure / Income / Transfers,
  each a pie + a two-line coloured header + an expandable breakdown tree) → full-width
  **Recurring** card → demoted monthly-**trend** strip. One `drill_down()` feeds all
  three columns (pie mirrors each column's list; header total is the branch node's
  `.amount`); `summary()` feeds only the Net strip. Explicit node→column map
  (Expenditure ← Spending node = `nodes[1]`, Income ← `nodes[0]`, Transfers ← `nodes[2]`)
  so a naive positional zip can't render Income inside the red Expenditure column.
  `__init__` gained a required `recurring: RecurringService`.
- **`ui/main_window.py`.** Constructs `RecurringService(self._service.vault)` and passes
  it to `HomeView(...)`, with `amount_prefs=` **by keyword** (the positional-shift the
  spec flagged — the new required arg lands before the defaulted `amount_prefs`).
- **Tests.** New `tests/features/dashboard_focus/` (spec.md + 20 legs covering INV-1..9 +
  the pie builder). Rippled the FIBR-0138 `dashboard_drilldown` and FIBR-0012 `dashboard`
  tests onto the new column surfaces (the retired `dashboard_drilldown` /
  `dashboard_category_*` / `dashboard_income` / `dashboard_expenditure` objectNames).
- **Docs.** `CHANGELOG.md [Unreleased]` + README "what works today" dashboard bullet.

No schema change (`LATEST_SCHEMA_VERSION` stays 9); no new service method; no new
dependency; no network.

## What was learned

- The recurring card uses **real `date.today()`** (the service has no `today` default),
  so its tests seed occurrences relative to `date.today()` and compare against a fresh
  `summary(date.today())` — deterministic on any calendar date, no clock injection at the
  widget seam.
- The Qt grid-rebuild for the recurring card needs `widget.setParent(None)` (not just
  `takeAt`) before repopulating, or `findChild` can return a stale figure label that is
  laid-out-removed but still parented.
