# FIBR-0015 — Windows self-contained `.exe` build

**Closed 2026-07-13 by `/close-phase`.** Kind: `chore` (build / ci / packaging).
Security: MEDIUM (swaps the package supplying the SQLCipher engine). Tag
`FIBR-0015-complete`.

## The blocker that wasn't

The roadmap recorded Windows as blocked because `sqlcipher3-binary==0.6.0` (the
pinned SQLCipher vault engine) ships **Linux/macOS wheels only** — so a Windows
`.exe` seemed to need a hand-compiled SQLCipher (Wine + MSVC, or a vendored DLL)
before anything else. That premise was **stale**: a sister distribution,
**`sqlcipher3-wheels`** (the `laggykiller/sqlcipher3` fork of `coleifer/sqlcipher3`),
publishes the **same project** with a Windows/macOS/Linux wheel matrix (incl.
`cp312-cp312-win_amd64`), exposes the **identical `sqlcipher3` import**, and bundles
the **identical SQLCipher 4.12.0 community** engine. So the Windows build reduced to
a dependency swap + a PyInstaller job on a `windows-latest` runner — no crypto
compilation. Proven empirically before the swap (bidirectional cross-package vault
round-trip + `sqlcipher_export`/`rekey` binding-surface spike); recorded in
**ADR-0009**.

## What was learned

- **A `pip install` premise can go stale silently.** The "Linux-only wheel"
  blocker was true for the *exact package pinned* but not for the project — a
  sister distribution already solved it. WebSearch even hallucinated a
  `-binary` 0.6.2 with Windows wheels; the PyPI JSON showed 0.6.0 was latest with
  no Windows wheel. The real answer (`sqlcipher3-wheels`) came from checking, not
  recalling (rule §13).
- **A dep swap surfaces hidden type debt.** `sqlcipher3-wheels` ships tighter
  types than `-binary` did, so `Cursor.lastrowid` became `int | None` (SQLite's
  true contract) instead of `Any` — breaking 7 repositories that returned it as
  `int`. Fixed at root cause with one shared `repositories.last_insert_id` helper
  (Rule of Three), not 7 `# type: ignore`s.
- **Stale local build artifacts bite mypy.** A months-old gitignored `build/lib/`
  and an `src/finbreak.egg-info/` (both from an old non-isolated `pip install .`)
  shadowed `src/finbreak` for mypy and named the old package; removed. CI is
  unaffected (fresh checkout + PEP 517 isolated builds).

## What was deferred

- **macOS `.dmg` + Flatpak/Flathub → FIBR-0130** (split at close per the
  FIBR-0127/8/9 precedent). The crypto blocker is already cleared for both by the
  same wheel, so that remainder is packaging-only.
- **Windows code-signing, MSI/installer, Windows auto-update** — deferred; testers
  use the unsigned onefile and update manually.
- **Rewiring the released bash freeze to import the canonical flag list** — a
  follow-up refactor; the Linux freeze stays the parity guard's read-source.
