# FIBR-0013 — P11 locked-PDF export with section selection (closed 2026-07-13)

## What changed

Shipped the password-protected PDF report export by TDD, in seven red→green
slices (`fa8bcc1`→`bff64c0`):

- **`services/pdf_export.py`** — `ExportOptions` + `PdfExportService`.
  `render_pdf_bytes(options, today=None) -> bytes` builds an HTML report
  (`QTextDocument`), embeds the donut + trend as **offscreen** `QChartView.grab()`

  `QPdfWriter(QBuffer)`, and — when a password is set — AES-256-encrypts the bytes

  (INV-2 structural). `export(options, out_path)` is the sole writer: temp `.part`

  unencrypted fallback (INV-2/INV-12). Report body: a header (title / generated
  line / human period label / `{names}` collapse > 3), an opt-in Summary (combined
  + per-account lines when > 1 account, ordered by name), Charts (Light/Dark
  `ChartTheme`), and a complete `(occurred_on, id)`-stable transaction list with an
  Account column only when > 1 account and confirmed transfers `⇄ Transfer`-marked.

  `build_donut_chart` / `build_trend_chart` (+ the relocated `_donut_wedges` and

## What was learned

- **`QChart` needs a full `QApplication`, not just `QGuiApplication`.** The
  offscreen rasteriser builds a `QChartView` (a QWidget); a spike under
  `QGuiApplication` segfaulted in `ChartThemeManager::decorateChart` →
  `QWidgetTextControl`. The service + its tests take the `qapp` fixture; the real
  bundle always has a `QApplication`.
- **All four spec "verify-empirically" Qt shapes held** on PySide6 6.11.1:
  `QPdfWriter` in `QtGui`, in-memory `QBuffer` PDF bytes, `addResource` image
  embedding, and `QChartView.grab()` offscreen — proven by a throwaway spike
  before any production code, so nothing shipped on an assumption (rule 13).
- **The venv has an installed `finbreak` shadowing `src/`.** `python -m finbreak
  --self-test` ran the *installed* copy (no `pdf_encrypt` leg) and misleadingly
  passed; the real check is `PYTHONPATH=src` (which is what the bundling test's
  subprocess uses).
- **`QPdfWriter` under the offscreen platform poisons a later SVG icon render**
  *only inside the shared pytest-qt process* — surfaced when running the bundling
  suite in isolation. The happy-path self-test already runs in a subprocess; the
  fix was to patch the new `_check_pdf_encrypt` to a no-op in the argon2/ofxparse
  FAIL tests (matching the existing "patch every prior check" pattern), so the
  real `QPdfWriter` never runs in-process there.
- **Export button must not `accept()`.** To keep the dialog open on a cancelled
  save or a failed write (D9), `ExportDialog` emits an `export_requested` signal
  (mirroring `SettingsDialog.saved`) instead of accepting; the shell closes it only
  on success.
