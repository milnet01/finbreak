# FIBR-0065 — Fix the auto-lock-during-modal-dialog crash (closed 2026-07-10)

**Kind:** fix. **Spec:** [`docs/specs/FIBR-0065.md`](../specs/FIBR-0065.md).
**Tag:** `FIBR-0065-complete`.

## What shipped

The reproduced **H-B** crash from the 2026-07-10 full-codebase sweep: an idle
**auto-lock** firing while a content-widget pop-up was open ran `MainWindow._lock()`
**inside** that dialog's nested `exec()` loop → `_clear_live()` →
`self._live.deleteLater()` destroyed the pop-up's parent mid-loop → the post-`exec()`
read (`dialog.selected_category_id()`, `dialog.password()`, …) touched a **deleted
C++ object** → `RuntimeError` the `VaultLockedError` guards couldn't catch.

**Approach (chosen with the user):** convert the six remaining blocking
`dialog.exec()` content-widget pop-ups to the shell's **non-blocking**
`setModal(True)`+`show()`+signal pattern (the same FIBR-0051 D2 adopted for the
shell's own dialogs), reading the result in an `accepted` slot. With no nested loop,
a lock destroys the pop-up **before** its slot can fire — no dead-object read, and
the slot (when it runs) always sees an unlocked vault.

- **`ui/modal.py`** (new) — `show_modal(dialog, on_accept)`: `setModal(True)` +
  `accepted→on_accept` + `finished→deleteLater` + `show()`. Six call sites.
- **home.py** — `_on_set_category` → `_apply_category` → `_maybe_offer_rule` →
  `_apply_learned_rule` (the set → offer → refresh chain, now async; refresh after
  set + again after a learned rule).
- **rules.py** — `_on_add`/`_apply_add`, `_on_edit`/`_apply_edit`.
- **statements.py** — `_on_reassign`/`_apply_reassign`.
- **import_wizard.py** — the **D5 state machine**: a shared
  `_try_decrypt(data, password) -> bytes | _NeedPassword | None` helper (the ONE
  place the non-password-error catch lives), `_begin_decrypt` (stored-once) →
  `_prompt_pdf_password` (non-blocking) → `_on_pdf_password` (re-prompt) →
  `_after_decrypt` (persist a verified password, both writes inside
  `if remember and password:`) → `_continue_after_decrypt` (the old `_select_pdf`
  tail). Preserves the FIBR-0009 (re-prompt / remember) / FIBR-0050 D6
  (decrypt-once) / FIBR-0057 (`_stored_pw` re-target carry) semantics exactly.
- Orphan `QDialog` imports removed (home/statements/import_wizard; rules keeps it —
  `RuleEditDialog(QDialog)`).
