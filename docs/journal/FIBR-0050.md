# FIBR-0050 — Standard Bank (SA) statement text-parser

Closed 2026-07-05. One text-layer reader for **all six** of the user's Standard
Bank account types (current, savings, home loan, revolving-credit, credit card,
money-market), family-dispatched inside a single module — explicitly **not** a
per-account-type importer. Extends P07 (FIBR-0009); adds no dependency and no
schema change.

## What changed

- **`src/finbreak/importers/standard_bank.py`** — the parser. `detect_standard_bank`
  gates on the SB legal marker (document-wide) + a family column-header signature
  over a sliding header-block window, most-specific-first C→D→B→A. Four pure
  line-grammar functions (`_parse_family_a/b/c/d`) share the number-format
  detection, year inference, continuation folding, integrity checksum, and
  `parse_transaction` machinery. `StandardBankImporter.parse` returns a
  `ParseResult` (or `None` → generic fallback) that feeds the existing
  `preview_result` → dedup → `commit_import` pipeline, skipping column-mapping
  like OFX.
- **Sign model (budget view):** balance families (A/B/D) import the printed
  magnitude carrying the running-balance-delta sign; the credit card flips its
  section-signed printed figure (INV-6). US `1,427.41` vs European `239.206,04`
  detected per-statement from the **transaction region** (D9).
- **Integrity (all-or-nothing, INV-11):** a per-row `|delta| == printed` gate
  (A/B/D) + a completeness gate against the independently-printed closing figure
  where one prints. Any failure raises **one** friendly `ValueError` and imports
  nothing (Savings, which prints no independent closing, rides the per-row gate).
- **`src/finbreak/ui/import_wizard.py`** — the FIBR-0050 D6 decrypt-once seam
  (`_decrypt_pdf`): a recognised SB statement is parsed and jumps to preview; any
  other PDF falls through to the generic table extractor.
- **`tests/features/standard_bank_pdf/`** — 36 tests (pure grammar helpers + `parse`
  per family + integrity + security + bounds + wizard round-trips). 13 committed
  **synthetic** reportlab fixtures (invented data, fake account number, the SB
  legal footer); encrypted variants made in-test with pikepdf. `reportlab` stays a
  probe/authoring tool, never a project/test dependency.

## What was learned

- The generic ruled-table extractor (FIBR-0009) genuinely can't read Standard
  Bank: the cheque statement's text has no grid lines (collapses to one cell), and
  the credit card interleaves two transaction columns per printed line. Reading
  the **printed text lines** is stable where the table geometry is not.
- **"Month-end Balance" is not the closing figure.** Probing the real statements
  showed Current uses "Balance at date of statement", RCP "Balance outstanding at
  date of statement", and Savings prints **no** independent closing at all — only
  a calendar "Month-end Balance" that must not be used as the checksum target.
- The Home Loan prints its running balance **unsigned** but its "CLOSING BALANCE"
  row **signed** — so `_verify_checksum` compares **magnitudes**, not values.
- `pikepdf.PasswordError` is **not** a subclass of `pikepdf.PdfError`; a corrupt
  file that passes the `%PDF-` sniff raises the base `PdfError`, which a
  `(ValueError, OSError)` net silently misses → a Qt-slot crash. The error net
  must name `PdfError` explicitly (caught in the confirming cold re-review).

## What was deferred

  this contract.
- Nothing was folded into `docs/known-issues.md`.

## Accepted limitations (verified, not defects)

- **`_capture_closing` uses a substring marker match.** In principle a summary
  line carrying both the marker and a money token (e.g. a hypothetical "Closing
  balance last statement 1,000.00" above the real ledger closing) could be picked
  as the closing figure. It is money-token-guarded (a pure-prose marker line is
  skipped), **fail-safe** (a wrong closing makes the completeness gate *reject* the
  statement, never silently under-import), and is **not present in any of the six
  real statements**. Surfaced by the cold re-review, accepted as a documented
  robustness limitation rather than risk an un-fixture-able change during close.
