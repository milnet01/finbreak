## What was learned

- **Empty-combo `currentData()` is `None`, which `findData(None)` matches to the
  Uncategorised item** — on first build the category filter silently defaulted to
  "Uncategorised" (categorised rows vanished). Guarded with a `count()`-check → `_FILTER_ALL`.
- **`_check_qtcharts` must not build a `QChart`** (a `QGraphicsWidget` needs a
  `QApplication`) — the self-test FAIL unit tests patch `_check_qt` away, so the leg
  uses a plain `QObject` `QBarSet` instead.
- The FIBR-0117 remembered-columns seam already saved/restored full header state
  (incl. section order) — enabling `setSectionsMovable` in that one shared helper lit
  up **drag-to-reorder + persistence** across every data table at once (shipped as
  FIBR-0120), a clean reuse rather than per-table code.

## Findings folded (indie-review, all fixed inline)

- **INV-2 year bound** — `report_prefs` bounded the month but not the year; a stored
  year outside 1–9999 would make `resolve_period`'s `date(...)` raise. Now downgrades
  to previous-month. +regression.
- **Column-key collision** — the Transactions + Statements tables (both 5-column,
  both unnamed) shared the empty `"columns/"` `remember_columns` key and
  cross-corrupted widths + drag order; each data table now has a distinct objectName.
  +regression.
- **Bare `except`** — `HomeView._on_period_changed` caught `Exception` and guarded
  only the write; now catches `VaultLockedError` specifically around the full
  persist+refresh in both selector slots (coding.md § 2).
