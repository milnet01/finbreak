> **At a glance:** INV-1 the self-test exercises Qt + SQLCipher + qpdf ·
> INV-2 both artifacts (onefile + AppImage) build · INV-3 they run in a
> Python-free container · INV-4 pinned toolchain (glibc floor ~2.36, build-image) ·
> INV-5 build stage opt-in so the everyday gate stays fast · INV-6 a
> fast pytest guards the self-test.

**Theme:** prove — *now*, on a trivial stub — that the three
**native** stacks the finished app depends on (Qt, an encrypted
SQLCipher database, and the qpdf library behind `pikepdf`) actually
travel inside a
downloadable artifact and run on a machine with **no Python
installed**. This surfaces ADR-0007's native-lib-collection risk at
P01, before ten feature phases are stacked on a packaging assumption
that might not hold.

## Context

ADR-0007 commits every release to being **fully self-contained**:
download one file, run it, nothing pre-installed. The hard part is not
the Python code — it is the **native** libraries: the Qt runtime +
plugins (PySide6), the SQLCipher library behind the encrypted vault
(ADR-0003), and the qpdf library behind `pikepdf` (ADR-0004). A bundle
that silently omits one of these runs on the build machine and fails on
the user's — the classic packaging failure ADR-0007 names.

This item builds a **one-command bundle-and-launch check** against a
throwaway stub that imports and exercises all three stacks, freezes it
into a **PyInstaller `--onefile`** binary *and* an **AppImage**, then
launches each inside a **Python-free container** and asserts it works.
That is ADR-0007's clean-machine exit criterion in miniature, runnable
today.

Scope decisions taken with the user before drafting (2026-07-01):

- **All three native stacks are de-risked now**, not just Qt +
  SQLCipher — the qpdf/`pikepdf` stack is the third native risk in
  ADR-0007's consequences list, so leaving it out would leave a known
  risk unproven.
- **The bundle-build is a separate, opt-in command** — it takes
  minutes (~2–5 min: venv provision + two freezes + two container
  launches), so it must not slow the everyday `ci-local.sh` gate (see
  INV-5).

### Decisions pulled forward from later phases

To exercise the native stacks, this item adds the app's first
**runtime** dependencies (previously only a `dev` tool group existed),
earlier than the P02 vertical slice that will first *use* them:

- **`sqlcipher3-binary`** is chosen as the SQLCipher Python binding —
  the maintained package whose wheel **bundles** the SQLCipher +
  crypto native code (no system `libsqlcipher`), which is precisely
  what lets it freeze cleanly. (`pysqlcipher3` is abandoned.) This is a
  **provisional** binding choice recorded here; the P02 crypto layer
  (FIBR-0004) formalises it in an ADR when the real Argon2id-keyed
  vault lands. The smoke-test uses an **obviously-fake literal key**
  (`"finbreak-smoke-test-not-a-real-key"`) purely to open a keyed DB —
  it is **not** real key derivation and is **not** a secret in the
  repo (security-model.md **INV-6** — no key/password/vault/real record
  ever committed, `gitleaks`-enforced; an obviously-fake literal is
  none of those and won't trip the FIBR-0001 secret-scan gate). INV-9
  (clean logs) is separately unaffected — the self-test logs nothing.

## Invariants (testable)

> These `INV-N` are **local to this spec**. They are separate from the
> project-wide security invariants in `docs/security-model.md § 5`;
> where this spec cites one it names the doc explicitly.

- **INV-1 — The self-test exercises all three native stacks.**
  `python -m finbreak --self-test` (a) constructs a Qt `QApplication`
  under `QT_QPA_PLATFORM=offscreen` (headless, no display); (b) opens a
  **SQLCipher**-encrypted temp-file database via `sqlcipher3`, sets the
  key with `PRAGMA key`, writes one row and reads it back; (c)
  constructs a `pikepdf.Pdf` in memory. On success it prints the exact
  sentinel line `FINBREAK_SELFTEST_OK` and exits 0; on the first
  failing stack it prints `FINBREAK_SELFTEST_FAIL: <stack>` (where
  `<stack>` is the lowercase binding token `qt` / `sqlcipher` /
  `pikepdf` — note the PDF stack's token is its **binding** name
  `pikepdf`, not the library name `qpdf`) and exits non-zero — a
  matchable prefix so a test can assert *which* stack broke, not just
  the exit code. The FAIL sentinel prints to **stdout** by design (so
  the container `grep -Fxq` scan and the diagnostic echo read one
  stream); the exit code is any non-zero, not contracted further. The
  entry point's three modes (all sentinels are exact lines on
  **stdout**):

  | Invocation | stdout line | Exit |
  |---|---|---|
  | `--self-test`, all stacks load | `FINBREAK_SELFTEST_OK` | 0 |
  | `--self-test`, a stack fails | `FINBREAK_SELFTEST_FAIL: <stack>` | non-zero |
  | no args (P01 has no GUI yet) | `FINBREAK_NOT_BUILT` | 0 |

  > **Superseded by FIBR-0004 (P02):** the `no args → FINBREAK_NOT_BUILT` row is
  > retired — no-args now launches the GUI. `--self-test` is unaffected (and now
  > covers a fourth native leg, Argon2).

  `--self-test` is a **permanent diagnostic mode** of the entry point
  (it survives as a way to diagnose a broken install on a user's
  machine), not throwaway scaffolding. Source: this spec; ADR-0007.
  Testable both in the dev venv (fast) and inside a frozen artifact
  (INV-3).

- **INV-2 — Both frozen artifacts are produced.**
  `scripts/build-smoke.sh` produces, from the `--self-test` entry
  point, two Linux x86-64 artifacts under `dist/`: a PyInstaller
  `--onefile` executable and an AppImage that **wraps that same
  onefile** (one freeze, then packaged into an AppDir — not a second,
  independent freeze). Both exist and are executable when the script
  exits 0. The build runs **inside a pinned `python:3.12-slim-bookworm`
  container** (not on the host): it apt-installs Qt's system-library
  deps, provisions a venv, installs the pinned deps, freezes both
  artifacts, and fetches `appimagetool` — all network-live. Building in a
  controlled older-glibc image (Debian 12, glibc ~2.36) rather than
  natively bounds the artifact's glibc floor to the container's, so the
  bundle runs on older targets than a bleeding-edge build host (glibc
  2.43) would allow — ADR-0007's old-base-image rule. (A `manylinux`
  image would give an even lower floor but ships a **static** Python that
  PyInstaller cannot freeze; `python:*-slim` provides the required
  shared `libpython`.) Only the two clean-room `run`s (INV-3) execute in
  the separate, offline (`--network=none`) test container. The build venv is
  **PySide6-only** — no second Qt binding (PyQt5/6, PySide2) — which
  the script asserts before freezing (the detection mechanism is
  implementer's discretion, e.g. a `pip list` check; on a second
  binding it exits non-zero with a diagnostic rather than freezing),
  satisfying PyInstaller 6.x's one-Qt-binding rule (INV-4). Source:
  this spec; PyInstaller 6.x behaviour.

- **INV-3 — The bundle runs with NO Python present (the de-risk).**
  Each artifact is launched inside a **Python-free glibc** container and
  must print `FINBREAK_SELFTEST_OK` and exit 0 — proving the CPython
  runtime and all three native stacks travel inside the artifact. The
  container run is locked down:

  - **Image:** a clean-room derived from `debian:13-slim` (a **pinned**
    release tag, not floating `stable-slim`, per global rule § 5a) with
    only the **universal desktop graphics baseline** added — `libGL` /
    `libEGL`. Every app-specific lib (CPython, Qt's glib/freetype/
    fontconfig/harfbuzz, SQLCipher, qpdf) travels in the bundle; `libGL`
    is the one lib PyInstaller (and every AppImage) leaves to the host —
    driver-tied and present on every real desktop — so adding it here
    represents a real GUI-capable machine, not a paper-over (see the
    disposition below).
  - **Environment:** `podman`/`docker run` do **not** inherit the host
    environment (the container env starts clean — there is no
    `--env-clear` flag, and none is needed), so the run passes only a
    fixed, container-sane
    `PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin`,
    `QT_QPA_PLATFORM=offscreen`, and `APPIMAGE_EXTRACT_AND_RUN=1`.
  - **Network:** none (`--network=none`).

  `build-smoke.sh`'s own exit is the AND of both launches: **0 only if
  both artifacts printed the sentinel and exited 0; non-zero if either
  failed.** The AppImage is launched with
  `--appimage-extract-and-run` (containers
  lack `/dev/fuse`); the PyInstaller onefile runs directly (it
  self-extracts to `$TMPDIR` / `/tmp`, which the scrubbed `PATH` does
  not affect). The container runner is `podman` or `docker` (identical
  `run` flags); the script **always prefers `podman` when present**,
  using `docker` only if podman is absent — CI relies on that same rule,
  not an environment-keyed choice. **Alpine is
  forbidden** as the test image — its musl libc cannot load the
  glibc-linked bundle. Source: ADR-0007 clean-machine exit criterion;
  PEP 656 (musl/glibc). **The offscreen-Qt launch surfacing a missing
  system library IS the de-risk working** — resolved by the correct
  disposition, never hidden: an **app-specific** Qt dependency (glib,
  freetype, fontconfig, harfbuzz, …) is **bundled** (installed in the
  build image so PyInstaller collects it into the artifact); the
  **driver-tied** graphics libs (`libGL`/`libEGL`), which PyInstaller
  deliberately excludes and every desktop provides, are the **one
  documented runtime requirement**, supplied by the clean-room graphics
  baseline above. Blanket-`apt install`-ing *app* libs into the test
  image to mask a real bundling gap remains forbidden — the split is
  deliberate. (Encountered for real 2026-07-01: the first launch died
  with `libGL.so.1: cannot open shared object file`; harfbuzz/glib/
  freetype/fontconfig were then bundled, `libGL` documented.) Such a
  loader failure typically crashes **before** any sentinel prints; the
  script therefore treats **absence of an exact `FINBREAK_SELFTEST_OK`
  stdout line** (matched with `grep -Fxq`, per the exact-line contract
  above) as failure (independent of exit code), so a crash-before-print
  is still caught by the AND rule above. When a launch fails, the script
  echoes that artifact's captured stdout — including any
  `FINBREAK_SELFTEST_FAIL: <stack>` line — in its diagnostic, so the
  failing stack is visible in the clean-room report.

- **INV-4 — Pinned bundling toolchain.** `pyproject.toml` pins the
  runtime deps and build tools **exactly** (`==`) to the versions in
  the table below (as researched 2026-07-01; **each `==` pin must be
  confirmed to resolve on PyPI at implementation time** — a wrong
  version fails the freeze) — the first three as `[project.dependencies]`,
  `pyinstaller` in a PEP 735 `build` dependency group:

  | Package | Pin | Role |
  |---|---|---|
  | `PySide6` | `==6.11.1` | Qt runtime (runtime dep) |
  | `sqlcipher3-binary` | `==0.6.0` | SQLCipher binding, native lib bundled in-wheel (runtime dep) |
  | `pikepdf` | `==10.9.1` | qpdf binding, `libqpdf` bundled in-wheel (runtime dep) |
  | `pyinstaller` | `==6.21.0` | freezer (`build` group) |

  The **effective minimum target glibc is the highest `manylinux_2_XX`
  floor across all three runtime wheels** (`PySide6`,
  `sqlcipher3-binary`, `pikepdf`) — the build reads and records the
  **max** of the three platform tags, not any single wheel's, and
  records that measured value. **Measured 2026-07-01:** the wheels' own
  floor is 2.34 (PySide6 6.11.1's `manylinux_2_34` wheel — the highest;
  `sqlcipher3-binary` is `manylinux2014` / glibc 2.17, `pikepdf`
  `manylinux_2_28`), but the artifact's *effective* floor is the **build
  image's glibc** — `python:3.12-slim-bookworm`, ~2.36 — because the
  bundled `libpython` links it. `debian:13-slim` (glibc 2.41) clears it.
  The build re-reads the max at build time in case a pin bump moves it. This is a
  build-time-verifiable figure, not sourced from ADR-0007 / design.md
  (which say only "old base image"). Recorded here so P13 release
  packaging inherits the floor. Exact pins serve reproducibility; the
  periodic sweep (coding.md § 1.5 / global rule § 5c) owns keeping them
  current, consistent with FIBR-0001 INV-4. Source: this spec;
  coding.md § 1.5 / global rule § 5 (latest-stable posture) + § 5c
  (sweep cadence).

- **INV-5 — The everyday gate stays fast; the build stage is opt-in.**
  `scripts/ci-local.sh` gains a build-smoke stage, but it is
  **skipped by default** and runs only when explicitly requested
  (`ci-local.sh --build`, or `FINBREAK_BUILD_SMOKE=1`); the default
  run keeps its seconds-level lint/format/security/test runtime. A
  **dedicated CI job** (manual `workflow_dispatch` + weekly schedule,
  **not** every push) runs the build stage. *Reconciliation:*
  FIBR-0001 INV-1 states FIBR-0003 "appends a build smoke-test stage to
  the same script" — this honours that literally (the stage is in the
  script). The opt-in gating keeps the fast path fast, and it is **not
  a silent drift** from that contract. The dedicated CI job's design
  (`workflow_dispatch` + weekly schedule) originates in **this spec +
  the user scope decision (2026-07-01)** — FIBR-0001 speaks only to
  appending a stage to the same script, not to a separate CI job.
  Source: FIBR-0001 INV-1/INV-2 (the stage-in-script reconciliation
  only); this spec + user decision (the CI-job design).

- **INV-6 — The self-test logic is guarded by a fast test.** A
  pytest that runs `python -m finbreak --self-test` **in the dev
  venv** (Python present) asserts the `FINBREAK_SELFTEST_OK` line is
  printed and the exit code is 0 — a fast regression guard on the three
  native imports that runs in the everyday gate, independent of the slow
  bundle build. The same test file also asserts the no-args mode prints
  `FINBREAK_NOT_BUILT` and exits 0; and it forces one stack to fail
  (monkeypatching a native import to raise) to assert the
  `FINBREAK_SELFTEST_FAIL: <stack>` line and a non-zero exit — so all
  three INV-1 entry-point modes have a guard. The
  full build-and-clean-room check is a separate `integration`-marked
  test that **skips** when the build tooling or a container runtime is
  absent (so the everyday gate never blocks on it); it asserts the
  script exits non-zero when a launch prints no `FINBREAK_SELFTEST_OK`,
  covering the crash-before-print path. Source: testing.md
  § 1 (TDD), § 3.3 (integration tests — the marker the slow
  build+clean-room test carries so it can be skipped).

### Failure modes (`build-smoke.sh`)

The standalone build script's non-happy paths are defined so an
implementer does not guess an exit code:

- **No container runtime on `PATH`** (neither `podman` nor `docker`):
  the script exits **non-zero** with a one-line diagnostic naming the
  missing runtime — it does **not** silently succeed, because the
  clean-room launch (INV-3) is the whole point. (The INV-6 `integration`
  pytest instead *skips* here, so the everyday gate never blocks.)
- **`appimagetool` fetch fails** (offline machine / upstream
  unavailable): the script exits **non-zero** naming the fetch failure.
  The **build** phase needs network to fetch tooling; only the **run**
  phase is `--network=none` (INV-3).
- **AppImage cannot extract-and-run in the container**
  (`--appimage-extract-and-run` — which bypasses FUSE by unpacking to a
  temp dir rather than mounting — itself errors): the script counts the
  AppImage as a **failed** artifact (non-zero exit, per INV-3's AND
  rule), not a pass.
- **Either artifact fails to *build*** (the PyInstaller freeze errors,
  or AppImage packaging fails) *before* the run phase: the script exits
  **non-zero** naming the artifact that failed to build and does not
  proceed to the clean-room launch. Any artifact that *did* build is
  left in `dist/` for inspection (not cleaned up). (INV-2's "both
  exist ... when the script exits 0" is the positive form of this rule.)
- **Both opt-in triggers are OR'd:** the build stage runs if *either*
  the `--build` flag **or** `FINBREAK_BUILD_SMOKE=1` is set (no
  precedence to resolve — either alone triggers it). Because
  `ci-local.sh` runs under `set -euo pipefail`, both are read with
  `set -u`-safe expansions (`${1:-}`, `${FINBREAK_BUILD_SMOKE:-}`) so
  the default no-arg fast path does not error on an unset variable.

## Deliverables

- `src/finbreak/__main__.py` — the entry point with the permanent
  `--self-test` mode; behaviour per the INV-1 table. Splitting the
  stack-exercising logic into a `_selftest.py` helper is the
  implementer's discretion (to keep `__main__` lean), not a required
  file.
- `pyproject.toml` — the first `[project.dependencies]` plus a `build`
  dependency group, both pinned per the INV-4 table; a glibc-floor note
  (the build-image glibc, ~2.36; wheels' own floor 2.34 — INV-4).
- `scripts/build-smoke.sh` — the opt-in bundle-and-clean-room command:
  builds inside a `python:3.12-slim-bookworm` container (PySide6-only
  venv), freezes both artifacts, fetches `appimagetool` and builds the
  AppDir + AppImage there, then runs both artifacts in the derived
  Python-free clean-room (debian:13-slim + graphics baseline) and asserts
  the sentinel
  (INV-2/INV-3). Re-runnable (a re-run overwrites prior artifacts and
  re-passes the sentinel check — the frozen binaries are not claimed to
  be byte-identical); artifacts land in `dist/` and fetched tools
  in `dist/.build-cache/` — both under `dist/`, which `.gitignore`
  already excludes (verified: it excludes `dist/`, `build/`, and
  `*.AppImage`), so nothing fetched leaks into the tree.
- `scripts/ci-local.sh` — append the opt-in build-smoke stage (INV-5).
- `.github/workflows/build-smoke.yml` — the dedicated CI job
  (`workflow_dispatch` + weekly `schedule`) on pinned `ubuntu-24.04`,
  running `ci-local.sh --build`.
- `tests/features/bundling/` — `spec.md` (the tiny test contract) +
  `test_bundling.py`: the fast dev-venv self-test guard (INV-6) and the
  `integration`-marked, tooling-gated full build+clean-room test.
- `CLAUDE.md` — "Build and test" documents `--self-test` and
  `scripts/build-smoke.sh`; "Module map" notes the new runtime deps.

## Out of scope

- **Windows / macOS** PyInstaller builds and the multi-OS build matrix
  — P13 (packaging & release; see [ROADMAP.md](../../ROADMAP.md)). This
  item is **Linux x86-64 only**.
- **Flatpak / Flathub** packaging — P13.
- **The full multi-arch / multi-OS reproducible release pipeline.** This
  smoke-test builds **one** Linux x86-64 bundle inside a
  `python:3.12-slim-bookworm` container — enough to floor its glibc at
  ~2.36 and prove it runs on `debian:13-slim` (glibc 2.41). P13 owns the complete pipeline: the
  Windows/macOS freezes, the Flatpak, and a reproducible pinned build
  base across every target. (A native host build was the original plan
  but was dropped when the smoke-test surfaced that this build host's
  glibc, 2.43, exceeds the `debian:13-slim` target — exactly the
  portability trap the de-risk exists to catch.)
- **Any real GUI window or feature** — P02+. The self-test constructs
  an offscreen `QApplication` and shows nothing.
- **The Qt SQL-driver and imageformat plugins** that ADR-0007 / design.md
  name as native-collection risks — this stub loads only the Qt
  **offscreen platform** plugin (SQLCipher is reached via `sqlcipher3`,
  not Qt's SQL driver; no image is decoded). Those plugins are exercised
  by the P02+ code that uses them; P13 packaging still bundles them. The
  three native **libraries** (Qt, SQLCipher, qpdf) are what this item
  de-risks.
- **Real key derivation** (Argon2id, FIBR-0004) and the real vault
  schema — the self-test's `PRAGMA key` uses a throwaway literal only
  to prove the encrypted-DB native path loads.
- **Code-signing / notarization** — P13.
- **`arm64` / other arches** — P13 if ever.

## Exit criteria (Definition of Done)

1. `python -m finbreak --self-test` prints `FINBREAK_SELFTEST_OK` and
   exits 0 in the dev venv; the fast guard test (INV-6) is green.
2. `scripts/build-smoke.sh` produces both artifacts, and **both** print
   the sentinel and exit 0 inside the Python-free `debian:13-slim`
   container (INV-2/INV-3).
3. The opt-in build stage exists in `ci-local.sh`, is **skipped** in
   the default fast run (INV-5), and the dedicated CI job runs it green.
4. The toolchain is pinned and the glibc floor (~2.36, the build-image
   glibc; the wheels' own floor is 2.34) recorded (INV-4).
5. `/audit` and `/indie-review` return zero **actionable** findings —
   non-allowlisted, severity ≥ LOW per `docs/audit-allowlist.md` — on
   the same closing pass. Audit scope = the files FIBR-0003 authors or
   modifies (the deliverables above).
6. User approves closing FIBR-0003.
