## 1. Problem & intent

finbreak ships today as a **single AppImage** (FIBR-0054) attached to the
GitHub release, plus a Windows `.exe`. That serves users who download-and-run,
but it is not a **first-class Linux install**: it does not appear in a
software centre, it is not on a distro update channel, and it needs the user to
find the GitHub release page, download the blob, `chmod +x`, and trust FUSE.

FIBR-0155 publishes finbreak through the **openSUSE Build Service** (OBS,
`build.opensuse.org`, where the user has an account). OBS builds **native
packages** (RPM for openSUSE/Fedora, deb for Debian/Ubuntu) from one source
recipe and publishes them to `software.opensuse.org` plus per-distro download
repos. The outcome the user wants: **install finbreak the normal way for your
distro, see it in your app menu / software centre, and get updates through your
distro's package manager** ‚Äî the "app store" experience.

This is **complementary** to, not a replacement for, the AppImage (FIBR-0054)
and the deferred Flatpak/Flathub route (FIBR-0130): AppImage = portable
download; OBS = distro-integrated install; Flatpak = sandboxed cross-distro.

## 2. Scope

**In scope**
- An **OBS project + package** (`packaging/obs/`, source-tracked in this repo)
  that builds native RPM + deb for the five concrete targets in ¬ß 3.1.
- A **PyInstaller-frozen payload** installed under `/usr/lib/finbreak/`
  (¬ß 3.2), with a **`/usr/bin/finbreak`** launcher (¬ß 3.4).
- **Reverse-DNS desktop integration** (¬ß 3.3): a checked-in
  `io.github.milnet01.finbreak.desktop`, the hicolor icon set, and an AppStream
  `io.github.milnet01.finbreak.metainfo.xml`.
- **Metainfo `<release>` mirrors CHANGELOG** (¬ß 3.7), synced by a small helper
  (changelog-writer-agent territory), so the store listing shows real notes.
- A **`[project.gui-scripts]` entry point** in `pyproject.toml` (¬ß 3.4) ‚Äî the
  documented, testable `finbreak.__main__:main` launcher target.
- **Offline, reproducible OBS builds** (¬ß 3.6): the pinned wheel closure is
  vendored (an `_service`-fetched source tarball + `pip download`), so the OBS
  build root needs no network.
- A **`security-model.md` note** that the distro build makes **zero** outbound
  requests by default (¬ß 3.8) ‚Äî the self-updater is inert (`detect_installer()`
  ‚Üí `None`), tightening INV-8 for this build.
- **Tests** by the FIBR-0096 script-scrape pattern (¬ß 4): assert the recipe /
  desktop / metainfo invariants without a real OBS build.

**Out of scope**
- **Getting into the official distro archives** (Fedora/Debian main). This is
  the author's **OBS home project** repo (`home:milnet01` or a devel project),
  the accepted venue for a bundled-runtime third-party app. Official-archive
  submission (which would force the distro-dep model ¬ß 7 rejects) is a much
  larger, separate effort.
- **Flatpak/Flathub** (FIBR-0130, deferred) and **macOS** packaging.
- **Sharing the security-critical native stack with distro libraries** ‚Äî see
  ¬ß 7 for why SQLCipher/qpdf/pdfium stay bundled (correctness + availability).
- **Auto-submitting releases to OBS from CI** ‚Äî the first cut is a documented
  manual `osc` submit tied to the tag; automation is a follow-up (¬ß 3.9).
- Changing the app's behaviour. The only source change is the additive
  `gui-scripts` entry point; everything else is packaging + docs.

### 3.2 Payload model ‚Äî a frozen runtime, not distro-shared deps

**Decision: the package installs the PyInstaller-frozen `--onedir` runtime tree
under `/usr/lib/finbreak/`**, exactly the payload the AppImage already bundles,
built **per target** inside OBS. It bundles its own Python ‚â• 3.12 and every
pinned native (SQLCipher, qpdf, pdfium, Qt) in-tree.

Why this and not the "purist" distro-dependency model (`Requires: python3-pyside6,
python3-pikepdf, python3-sqlcipher, ‚Ä¶`):

1. **Correctness (decisive for this app).** finbreak is correctness-critical ‚Äî
   it encrypts people's money data. `sqlcipher3-wheels` is a **specific fork**
   bundling SQLCipher 4.12.0 with `SQLCIPHER_COMPAT = 4`; substituting a
   distro `python3-sqlcipher3` binding (different upstream, possibly different
   SQLCipher version / KDF defaults) risks **vault incompatibility** and is not
   something to discover in the field. The frozen payload is **byte-identical
   to the gate-tested app** ‚Äî same pins, same natives, same Python.
2. **Availability.** `pypdfium2` (Google's pdfium, via pdfplumber) is **not
   reliably packaged** across Fedora/openSUSE/Debian; the `laggykiller`
   SQLCipher fork isn't distro-packaged at all. A distro-dep model would be a
   hard **blocker** on at least one target and a per-family dep-name matrix to
   maintain on the rest.
3. **Python floor.** finbreak needs **‚â• 3.12**; Leap 15.6, Debian 12, and
   Ubuntu 22.04 default to older. A bundled Python removes the floor problem
   entirely, keeping older-stable targets in the matrix (a venv-on-system-python
   model, ¬ß 7, would drop them).
4. **Reuse.** It reuses the **existing, gated** `build-smoke.sh` freeze
   (rule 3) ‚Äî minimal new build surface.

The accepted cost is a **larger, self-contained package** that does not share
libraries with the distro and does not receive distro security updates for its
bundled deps. That is the standard, accepted tradeoff for a **third-party OBS
home-project** app (it is not entering the official archive, ¬ß 2), it is
already how the AppImage ships, and the **FIBR-0096 SBOM** discloses the bundled
closure per release. Distro-native **update delivery still works** ‚Äî a new
finbreak version is a new package in the OBS repo, pulled by
`zypper up`/`apt upgrade`.

This is **not** "wrap the AppImage" (the rejected option): there is no `.AppImage`
blob, no FUSE. The RPM/deb lays the freeze out in the native `/usr` filesystem
with a real launcher, `.desktop`, icons, and metainfo ‚Äî a proper distro package
whose payload happens to be a frozen runtime.

### 3.3 Desktop integration ‚Äî reverse-DNS app-ID

Three checked-in assets (source of truth in `packaging/obs/`), installed to the
standard XDG locations by the RPM/deb:

- **`io.github.milnet01.finbreak.desktop`** ‚Üí `/usr/share/applications/`.
  Per `naming.md:175-177`. Key fields:
  - `Exec=finbreak` (the ¬ß 3.4 launcher; **not** an absolute freeze path).
  - `Icon=io.github.milnet01.finbreak` (themed icon name = app-ID).
  - `StartupWMClass=finbreak` ‚Äî **must equal the runtime WM_CLASS the app sets**
    (`app.py:38`), which is the bare `finbreak`, **not** the app-ID. This is the
    subtle correctness point: get it wrong and the taskbar shows a duplicate
    generic icon instead of grouping under the launcher.
  - `Categories=Office;Finance;` (a personal-finance app; the AppImage's default
    `Utility;` is wrong for a software-centre listing).
  - `Terminal=false`, `Type=Application`.
- **Hicolor icons** ‚Üí `/usr/share/icons/hicolor/<size>x<size>/apps/io.github.milnet01.finbreak.png`
  for each size present in `assets/icon/finbreak-{16,24,32,48,64,128,256,512}.png`
  (renamed to the app-ID at install). Enables the software-centre + menu icon.
- **`io.github.milnet01.finbreak.metainfo.xml`** ‚Üí
  `/usr/share/metainfo/`. AppStream component (`desktop-application`) with:
  `<id>io.github.milnet01.finbreak</id>`, name, summary, a `<description>`,
  `<metadata_license>` (CC0-1.0 for the metainfo) + `<project_license>MIT</project_license>`,
  `<url type="homepage">`, `<launchable type="desktop-id">io.github.milnet01.finbreak.desktop</launchable>`,
  screenshots, and a `<releases>` block (¬ß 3.7). It **must pass**
  `appstreamcli validate` (INV-4) ‚Äî this is what software.opensuse.org and GNOME
  Software / KDE Discover consume for the listing.

### 3.4 The `/usr/bin/finbreak` launcher + entry point

Two coordinated pieces:

1. **`pyproject.toml`** gains
   `[project.gui-scripts]` ‚Üí `finbreak = "finbreak.__main__:main"`. This is the
   canonical, **testable** declaration of the launch target (a test asserts it
   resolves to `__main__.main`), and it is what a from-wheel install would use.
2. **The packaged `/usr/bin/finbreak`** is a tiny shell wrapper the RPM/deb
   installs, `exec`-ing the frozen entry inside `/usr/lib/finbreak/` (the
   onedir's bootloader), **not** a `#!/usr/bin/python3` script ‚Äî because the
   payload is frozen, not a site-packages install (¬ß 3.2). It passes `"$@"`
   through, so `finbreak --self-test` works from the installed package (INV-2,
   reusing the FIBR-0003 native-stack proof as the package's post-install smoke).

`main()` already rejects unknown args with exit 2 and handles `--self-test`
(`__main__.py:17-34`) ‚Äî no change needed there.

### 3.5 The RPM `.spec` and `debian/` recipes

Both drive the **same three steps**: (1) freeze the payload, (2) lay it out
under `/usr/lib/finbreak/`, (3) install the ¬ß 3.3 desktop assets + ¬ß 3.4
launcher. Version comes from `__version__` via the `_service` (¬ß 3.9), never
hand-typed (INV-6).

- **`packaging/obs/finbreak.spec`** (RPM ‚Äî openSUSE + Fedora):
  - `Version:` injected by the OBS `set_version` service from the tarball.
  - `BuildRequires:` python ‚â• 3.12, `pip`, PyInstaller's needs, and the Qt/GL
    runtime libs from `ci-setup.sh:32-34` **translated to RPM names**
    (Mesa `libGL`/`libEGL`, `glib2`, `libxkbcommon`, `dbus-1`,
    `fontconfig`/`freetype`) ‚Äî the single build-dep list, mirrored per family.
  - `%build`: `pip install --no-index --find-links vendor/ .` into a build venv,
    then the PyInstaller freeze (reusing the build-smoke flags).
  - `%install`: copy the onedir tree to `%{buildroot}/usr/lib/finbreak/`, the
    launcher to `/usr/bin/finbreak`, and the desktop/icon/metainfo assets.
  - `%files`: the three XDG assets + the payload tree + launcher.
  - **No `Requires:` on the bundled natives** (they are in-tree) ‚Äî only a
    minimal runtime set the freeze genuinely dlopens from the system
    (the ¬ß 3.1 Qt libs), matching what the AppImage needs at runtime.
  - `%check`: run `finbreak --self-test` against the staged freeze (the FIBR-0003
    sentinel) ‚Äî fail the build if the native stack won't load.
- **`packaging/obs/debian/`** (deb ‚Äî Debian + Ubuntu): `control`
  (`Build-Depends:` = the same Qt/GL libs as apt names, already known from
  `ci-setup.sh:32-34`), `rules` (the same freeze‚Üílayout‚Üíinstall steps via
  `dh`), `changelog` (generated from CHANGELOG, ¬ß 3.7), `install`, `finbreak.desktop`
  hookless install. `debian/rules` runs the same `--self-test` gate.

### 3.6 Offline, reproducible OBS builds ‚Äî vendored wheels

OBS build roots are **network-isolated**. The pinned wheel closure is therefore
**vendored**, not fetched during the build:

- The OBS package's **`_service`** (¬ß 3.9) fetches the tagged source tarball
  (`obs_scm`/`tar_scm` on `v{VERSION}`) **and** the exact wheel closure
  (`pip download` of the `pyproject.toml` pins for each target's
  platform tag, into `vendor/`). The build then does
  `pip install --no-index --find-links vendor/` ‚Äî **zero network**, and the
  installed versions are provably the pinned ones (INV-7).
- This makes each target's build **reproducible** from the tag: same source,
  same wheels, freeze produced on that target's toolchain (so it links the
  right libc/Qt for that distro ‚Äî a genuine per-target-native advantage over the
  one-glibc AppImage).

### 3.7 AppStream `<release>` mirrors CHANGELOG

The metainfo `<releases>` block and the `debian/changelog` are **generated from
`CHANGELOG.md`**, not maintained by hand (drift is the failure mode the whole
FIBR-0096/`bump.json` lockstep discipline exists to prevent):

- A helper (`scripts/gen-metainfo-release.py`, changelog-writer-agent territory)
  reads the current dated CHANGELOG section and emits the top
  `<release version="{VERSION}" date="{DATE}">` with a `<description>` of the
  section's bullets. Run at release time (hooked into `/bump` alongside the
  existing lockstep edits), so cutting a release updates the store notes.
- INV-4's validation includes: the metainfo's newest `<release version>` equals
  `__version__`.

### 3.8 security-model.md impact ‚Äî zero outbound in distro builds

The distro package is launched from `/usr/bin/finbreak` with **no `$APPIMAGE`
env** and is **not** a frozen Windows exe, so `detect_installer()` ‚Üí `None`
(¬ß 3.1): the self-updater is **inert** and the Settings "check for updates"
checkbox is **disabled**. Therefore the distro build makes **zero** outbound
requests by default ‚Äî a **strict subset** of INV-8's "exactly one opt-in flow".

- This is covered **without a source change** by the existing runtime gating.
  The spec adds a **note** under INV-8 (`security-model.md`) recording that the
  distro build's outbound surface is empty, and a **test** (¬ß 4) asserting
  `detect_installer()` returns `None` when `$APPIMAGE` is unset and `sys.frozen`
  is falsy (belt-and-suspenders: proves a future refactor can't silently arm
  the updater in a distro install).
- No new networking dependency is introduced; the package inherits the
  no-network-import allowlist unchanged.

### 3.9 Version / release-automation tie-in

- **Version flows one way:** git tag `v{VERSION}` ‚Üí OBS `_service`
  (`tar_scm` `versionformat` / `set_version`) ‚Üí `.spec`/`debian` `Version:`.
  Never hand-entered (INV-6).
- **First cut = documented manual submit.** After a release is tagged + the
  GitHub release exists, the maintainer runs a documented `osc` flow
  (`osc service run` to refresh vendored sources, `osc commit`) to trigger the
  OBS rebuild. **CI auto-submit to OBS is out of scope** (¬ß 2) ‚Äî a follow-up
  once the manual path is proven on a real build.
- The `/bump` recipe gains the metainfo/changelog sync step (¬ß 3.7) so a bump
  keeps the store notes in lockstep with the other five version-bearing files.

## 4. Invariants

Tested by script-scrape (FIBR-0096 pattern) ‚Äî no real OBS build in CI.

- **INV-1 ‚Äî Frozen payload, not distro-shared security stack.** The recipes
  install a self-contained payload under `/usr/lib/finbreak/`; neither the
  `.spec` `Requires:` nor `debian/control` `Depends:` names a distro SQLCipher,
  qpdf, pdfium, or PySide package (the security-critical stack stays bundled,
  ¬ß 3.2). *Test:* grep the recipes; assert absence of those dep names.
- **INV-2 ‚Äî Installed launcher works + self-tests.** `/usr/bin/finbreak` exists,
  is executable, `exec`s the frozen entry with `"$@"` passthrough; a `%check` /
  `debian/rules` step runs `finbreak --self-test` (FIBR-0003 sentinel).
  *Test:* scrape both recipes for the launcher install + the `--self-test` gate.
- **INV-3 ‚Äî Reverse-DNS app-ID + correct WM class.** The `.desktop` basename,
  `Icon=`, metainfo `<id>`, and installed icon names are all
  `io.github.milnet01.finbreak`; `StartupWMClass=finbreak` (the runtime
  WM_CLASS, `app.py:38`), `Exec=finbreak`. *Test:* parse the `.desktop` +
  metainfo; assert each field, and assert `StartupWMClass` == the value
  `app.py` sets (read from source, so a runtime change breaks the test).
- **INV-4 ‚Äî Metainfo validates + mirrors version.** The metainfo passes
  `appstreamcli validate` (or `appstream-util validate-relax` fallback), and its
  newest `<release version>` equals `__version__`. *Test:* run the validator if
  present (skip-if-absent, like optional-tool gates); always assert the version
  match by reading `__init__.py`.
- **INV-5 ‚Äî Self-updater inert in a distro launch.** With `$APPIMAGE` unset and
  `sys.frozen` falsy, `detect_installer()` returns `None` and
  `is_update_supported()` is `False`. *Test:* monkeypatch env/`sys.frozen`,
  assert both.
- **INV-6 ‚Äî Version single-source.** No recipe hand-codes a version literal; the
  `.spec`/`debian` version derives from the tag via `_service`. *Test:* grep the
  recipes for a hard-coded `Version:` digit-string; assert it is the service
  placeholder, not a literal.
- **INV-7 ‚Äî Offline build, pinned closure.** The build step uses
  `pip install --no-index --find-links vendor/`; the vendored set matches the
  `pyproject.toml` pins. *Test:* scrape the recipe/`_service` for `--no-index`;
  assert no `pip install` without `--no-index` in the build phase.
- **INV-8 ‚Äî Entry point declared.** `pyproject.toml` `[project.gui-scripts]`
  maps `finbreak` ‚Üí `finbreak.__main__:main`, and that attribute is importable
  + callable. *Test:* load the project metadata; import + assert.

## 5. Exit criteria (falsifiable)

- A real OBS project builds **green** for all five targets in ¬ß 3.1
  (Tumbleweed, Leap 15.6, Fedora latest, Debian 13, Ubuntu 24.04).
- On a clean VM: install from the OBS repo (`zypper`/`dnf`/`apt`) ‚Üí finbreak
  appears in the desktop menu / software centre with its icon; launches;
  `finbreak --self-test` prints the FIBR-0003 sentinel.
- `appstreamcli validate io.github.milnet01.finbreak.metainfo.xml` passes.
- `software.opensuse.org` search shows finbreak with the CHANGELOG-derived notes.
- In the installed app, Settings ‚Üí "check for updates" is **disabled** (no
  outbound flow); a network capture during a normal session shows **zero**
  requests.
- The full local gate (`ci-local.sh`) stays green with the new tests + the
  additive `gui-scripts` entry (`-m "not perf"`).

## 7. Alternatives considered

- **Distro-shared dependencies** (`Requires: python3-pyside6, python3-pikepdf,
  python3-sqlcipher, python3-pdfplumber, ‚Ä¶`; app installed as a noarch wheel).
  *Rejected:* the security-critical SQLCipher binding is a **specific fork**
  (correctness risk to swap, ¬ß 3.2.1); `pypdfium2`/pdfium is **not reliably
  packaged** (blocker, ¬ß 3.2.2); version skew fights the deliberate pins; and it
  needs distro python ‚â• 3.12, dropping Leap 15.6 / older LTS. This is the model
  the **official distro archive** would demand ‚Äî hence archive submission is
  explicitly out of scope (¬ß 2).
- **Bundled venv on the distro's system python3** (ship a private venv of the
  pinned wheels, `#!/usr/bin/python3.12` launcher). *Rejected:* reintroduces the
  **Python-floor** fragmentation (needs each target to ship ‚â• 3.12; drops
  Leap-default / Debian 12 / Ubuntu 22.04), and still must bundle the in-wheel
  natives anyway ‚Äî so it carries the frozen-payload cost **without** the
  universality. PyInstaller-onedir (¬ß 3.2) bundles Python once and works
  everywhere.
- **Wrap the frozen AppImage in an RPM/deb** (the option the user explicitly
  rejected). *Rejected by user decision;* also technically inferior for a store
  listing (FUSE dependency, a blob under `/opt`, no clean `/usr` layout).
- **Flatpak/Flathub instead** (FIBR-0130). *Deferred, complementary* ‚Äî a
  different distribution channel (sandboxed, cross-distro), not a substitute for
  native distro packages + the openSUSE portal the user asked for.
