# FIBR-0054 — Optional in-app auto-update (Linux / AppImage first)

> **Status:** ✍️ **DRAFT — NOT YET CLEARED FOR CODE.** Awaiting the `/cold-eyes`
> loop (global rule § 14 — loop until a cold pass returns zero verified actionable
> findings). Do **not** start TDD until the Cold-eyes loop log below records
> convergence.
> **Phase:** near-term user request (the last one before P09/FIBR-0011).
> **Kind:** feature.
> **Source:** user-request-2026-07-09 (offer, never force, updates) + design
> brainstorm 2026-07-10 (scope decided with the user: **full in-app download +
> install + relaunch, Linux/AppImage first**; a **Windows** installer is a future
> plug-in behind the same seam, *designed for* but **not built** here; the
> integrity gate is an **Ed25519 signature**).
> **Security: HIGH — this is the app's first-ever network egress AND it installs
> executable code.** Two brand-new attack surfaces the FIBR-0004 security spine
> never had: (1) an **outbound network call** (security-model.md INV-8 currently
> says the shipped app "opens no sockets and makes no outbound call of any kind"),
> and (2) **running a downloaded binary**. Both are gated hard: the call is
> **opt-in, off by default**, and the download is **never executed unless an
> Ed25519 signature over its exact bytes verifies against a public key baked into
> the app**. This spec amends security-model.md INV-8 to record the single,
> consented exception and keeps every other network module banned.
> **Roadmap:** [ROADMAP.md → FIBR-0054](../../ROADMAP.md).
> **Governed by:** `docs/standards/{coding,naming,dependencies,testing,commits}.md`
> (`naming.md § 4` governs the new module names `services/update.py`,
> `services/update_fetch.py`, `services/update_installer.py`,
> `services/update_key.py`, `ui/update_dialog.py`, `ui/_update_worker.py`;
> `dependencies.md` governs promoting `cryptography` from a transitive to an
> explicitly-declared runtime dependency; `coding.md § 5.2` the `tr()` / RTL
> obligations of the new dialog);
> [ADR-0007](../decisions/0007-packaging-pyinstaller-appimage-flatpak.md)
> (the AppImage/packaging story this consumes + extends);
> [ADR-0003](../decisions/0003-sqlcipher-local-only-storage.md) (the vault whose
> key must be wiped before relaunch).
> Builds on [FIBR-0003](FIBR-0003.md) (the PyInstaller + appimagetool bundling
> machinery Phase 1 reuses to produce a real release AppImage),
> [FIBR-0051](FIBR-0051.md)/[FIBR-0052](FIBR-0052.md) (the `QMainWindow` shell +
> its plaintext `window.ini` + tracked-dialog patterns), [FIBR-0055](FIBR-0055.md)
> (the Settings dialog this adds one checkbox to), and
> [FIBR-0065](FIBR-0065.md) (the non-blocking `ui/modal.py::show_modal` pattern the
> update prompt uses instead of `exec()`).

**Theme:** the roadmap bullet asks for an **optional** updater: on a newer version,
prompt the user with **Later** (re-ask next launch) / **Skip this version** (never
re-prompt for it) / **Update now** (download → install → relaunch). We build the
**full flow for Linux AppImage** — the one platform where the install hand-off is
simplest (the running AppImage knows its own path via `$APPIMAGE`, so "install" is
*replace one file and re-exec*) and where the user can dogfood it end-to-end on
their own machine. The feature ships in **two phases** because the updater cannot
be built or tested until a real, signed release exists to update *from*:

- **Phase 1 — a real Linux release.** Give the app a real version (`0.1.0`), build
  a **real** release AppImage (not the FIBR-0003 self-test stub), sign it
  (Ed25519), and publish GitHub Release `v0.1.0` with the AppImage + its signature.
  This is the Linux slice of P13/FIBR-0015 pulled forward, and reusable there.
- **Phase 2 — the in-app updater.** The opt-in check, the prompt, the
  signature-verified download, the atomic swap, and the relaunch — built and tested
  against Phase 1's release.

## At a glance — invariants

| INV | Guarantee | Upholds |
|-----|-----------|---------|
| INV-1 | **(Security.) Opt-in, off by default — the check is the *only* deliberate network egress.** No outbound call happens unless the plaintext flag `check_for_updates` is `true` in the `window.ini` sibling (`paths.window_settings_path()`). An **absent** key ⇒ off (a fresh install never phones home). `UpdateService.is_enabled()` returns `False` for absent/`"false"`/malformed. A test asserts: absent → no check runs (the injected fetcher is never called); explicitly enabled → it is. | security-model INV-8 (amended: this is the single consented exception) |
| INV-2 | **(Security.) The check needs no vault and runs while locked, touching no decrypted data.** The launch check reads only the plaintext INI, the compiled-in `__version__`, and the public GitHub Releases API — never the vault key, never a decrypted row. It runs on the still-locked shell (before or without any unlock). A test drives the check with a locked `AuthService` and asserts it completes and never calls a vault-connection accessor. | FIBR-0004 INV-3 (key lifetime); security-model INV-1 |
| INV-3 | **A newer version is offered only for a strictly-greater, well-formed, non-skipped tag (fail-safe parse).** `check_for_update()` parses the release `tag_name` (a leading `v` stripped) and the current `__version__` into integer tuples via a dependency-free parser; it returns an `UpdateInfo` **only if** `latest > current` **and** `latest` is not the skipped version **and** an AppImage asset **and** its `.sig` asset are both present. A malformed `tag_name`, a malformed `__version__`, a missing asset, or `latest <= current` ⇒ returns `None` (no offer). Tests: newer→offer; equal→None; older→None; malformed tag→None; missing `.sig` asset→None; skipped==latest→None. | roadmap bullet (offer, don't force) |
| INV-4 | **(Security — the core gate.) Only an Ed25519-signed download is ever installed.** `download_and_verify(info)` downloads the AppImage **and** its detached `.sig`, then verifies the signature over the **exact downloaded bytes** with `cryptography`'s `Ed25519PublicKey.verify`, using the **public key committed at `services/update_key.py`**. On **any** verification failure (bad signature, truncated file, wrong key, absent `.sig`) it deletes the temp file and raises `UpdateVerificationError` — **nothing is installed**. A test signs a blob with a throwaway private key whose public half is monkeypatched in → verify passes; a one-byte tamper of the blob **or** the signature → `UpdateVerificationError` and the temp file is gone. | security-model INV-2/INV-8; ADR-0007 |
| INV-5 | **The swap is atomic; the running binary is never left half-written.** The download lands in a temp file **in the same directory as `$APPIMAGE`** (same filesystem, so the rename is atomic); only **after** INV-4 passes does the installer `chmod 0o755` it and `os.replace(temp, appimage_path)`. Any failure before the `os.replace` leaves the original `$APPIMAGE` byte-for-byte intact (the temp is deleted). A test points `AppImageInstaller` at a fake `$APPIMAGE` temp file, applies a new file, and asserts the target now holds the new bytes and is executable; a pre-replace raise leaves the original untouched. | ADR-0007 |
| INV-6 | **(Security.) The derived key is wiped before the relaunch `exec`.** `os.execv` replaces the process image and does **not** run Qt's `aboutToQuit`, so the installer path explicitly calls the key-wipe (`AuthService.on_about_to_quit`) **before** `os.replace` + `os.execv`, so no decrypted key survives into the relaunched process. A test asserts the wipe hook is invoked before the (monkeypatched) `execv`. | FIBR-0004 INV-3 |
| INV-7 | **(Security/robustness.) Off an AppImage the whole feature is inert.** `detect_installer()` returns an `AppImageInstaller` **only** when `$APPIMAGE` is set and points at an existing file; otherwise `None`. With no installer: no launch check runs, no prompt appears, and the Settings checkbox is **disabled with an explanatory tooltip**. So a `python -m finbreak` run, a Flatpak, or a future un-wired Windows build never half-updates. Tests: `$APPIMAGE` unset → `detect_installer() is None` and `is_update_supported()` is `False`; the Settings checkbox is disabled. | ADR-0007 (per-platform mechanism) |
| INV-8 | **Skip persists; Later does not; neither carries sensitive data.** "Skip this version" writes `skipped_update_version = <version>` to the plaintext INI; a subsequent check for that same version returns `None` (INV-3). "Later" writes nothing — the next launch re-checks and re-prompts. Both keys live in `window.ini` (never the vault) and hold only a version string / a boolean. A test: skip `0.1.1`, re-run the check against a release advertising `0.1.1` → `None`; against `0.1.2` → offered. | roadmap bullet (Later/Skip semantics) |
| INV-9 | **(Security.) The prompt + progress are non-blocking and auto-lock-safe.** `UpdateDialog` is shown via `ui/modal.py::show_modal` (`setModal(True)` + `show()` + a signal) — **never** `exec()`. If an idle auto-lock fires while it is open (only possible when unlocked), the FIBR-0065/FIBR-0051 teardown destroys it with the rest of the tracked chrome; the update path holds no vault reference, so nothing crashes. A test (mirroring the FIBR-0065 dialog-lifecycle grep) asserts `ui/update_dialog.py` contains no `.exec(`. | FIBR-0065 INV-1; FIBR-0051 INV-4b |
| INV-10 | **(Security.) The download is resource-bounded.** The fetch is capped by a maximum byte size (`_MAX_UPDATE_BYTES`) and a socket timeout; a response exceeding the cap is abandoned (temp deleted, `ValueError`) before it can exhaust disk, and a stalled connection times out rather than hanging the check. Mirrors the INV-5b resource-cap pattern (FIBR-0041 / security-model INV-5b). A test feeds an oversized stream and asserts the abort. | security-model INV-5b |
| INV-11 | **Any network/API failure is silent and safe.** A DNS/connection error, a non-200 status, a timeout, or unparseable JSON during the **check** is caught, logged (no traceback to the user), and results in **no prompt** — the app proceeds exactly as if up to date. Only an explicit **Update now** surfaces an error dialog (INV-4/INV-5/INV-10 failures) and stays on the current version. A test makes the injected fetcher raise and asserts `check_for_update()` returns `None` without propagating. | roadmap bullet (never nag); security-model (no-network default preserved on failure) |
| INV-12 | **(Security.) Network code is confined to one allowlisted module; the ban holds everywhere else.** The **only** file under `src/finbreak/` permitted to import `urllib` is `services/update_fetch.py`; `socket`, `http`, `requests`, and `ftplib` stay banned in **every** file, including that one (`urllib.request` over the default-TLS `https://` endpoint is the entire network surface). `test_INV8_no_network_imports_under_src` is amended to encode exactly this allowlist (one filename, `urllib` only) and still fails on any other network import anywhere. | INV-13 | **(i18n.) Every visible string in the new dialog is translatable; the dialog is RTL-safe.** All `UpdateDialog` labels/buttons/title use `self.tr("literal")` (no `self.tr(variable)`); the new Settings checkbox text is a `tr()` literal. | INV-14 | **(Phase 1.) ## Context

**The app has no version and no releases yet.** `src/finbreak/__init__.py` has
`__version__ = "0.0.0"`; `gh release list` is empty; there is no `bump.json`. Phase
1 establishes the first real version + release.

**The network ban is real and tested.** `docs/security-model.md` INV-8 states the
shipped app "opens **no sockets and makes no outbound call of any kind**," enforced
by `tests/features/vault/test_vault.py::test_INV8_no_network_imports_under_src`
(an `ast` walk over **every** `*.py` under `src/finbreak/`, banning
`_BANNED_NETWORK = {"socket", "http", "urllib", "requests", "ftplib"}` as static
`import`, `from`, and dynamic `__import__`/`import_module(...)` forms) plus
`test_INV8_no_network_package_in_runtime_deps` (no banned **distribution** in
`pyproject` runtime deps). `ofx_import` has a module-scoped sibling
(`test_INV8_no_network_import_in_ofx_module`). Adding the updater is the deliberate,
consented breach of INV-8, so both the doc and the src-scan test are amended
(D10/D12).

**The AppImage build machinery exists (FIBR-0003).** `scripts/build-smoke.sh` +
`scripts/_build-smoke-in-container.sh` already freeze `python -m finbreak` with
PyInstaller and wrap it into `finbreak-selftest-x86_64.AppImage` via `appimagetool`
inside `python:3.12-slim-bookworm`, then launch it in a Python-free `debian:13-slim`
container. Today they build the `--self-test` stub; Phase 1 adds a sibling that
freezes the **real GUI** entry.

**The shell + reuse points (verified 2026-07-10):**

- `app.run()` (`app.py`) builds the `QApplication`, constructs `MainWindow(service)`,
  `window.show()`, `app.exec()`. `app.aboutToQuit.connect(service.on_about_to_quit)`
  wipes the key on a normal quit.
- `MainWindow.__init__` builds chrome/content/status bar, restores geometry, sets
  `service.on_auto_lock = self._lock`, then routes to first-run/unlock. This is the
  natural place to arm the launch check (after construction, via
  `QTimer.singleShot(0, ...)` so it runs once the event loop is up and the — still
  locked — window is shown).
- `MainWindow._settings()` returns
  `QSettings(str(paths.window_settings_path()), QSettings.Format.IniFormat)` — the
  plaintext INI the opt-in + skip-version keys live in.
- `MainWindow._open_dialog(dialog, defer=False)` / `_teardown_dialog()` /
  `_lock` (step 1 tears down `self._dialog`, step 5 re-opens the unlock dialog) —
  the tracked-modal lifecycle; `ui/modal.py::show_modal(dialog, on_accept)`
  (FIBR-0065) is the non-blocking helper.
- `MainWindow._open_url(url)` = `QDesktopServices.openUrl(QUrl(url))` — the
  user-initiated browser egress used for the Donate links; the "what's new" release
  link reuses it (opening the browser is **not** an app socket).
- `ui/_worker.py::DeriveWorker(QThread)` — the parented, `finished→deleteLater`,
  `.start()` worker pattern the `UpdateCheckWorker` mirrors.
- `ui/settings.py::SettingsDialog(service, base_currency, parent)` +
  `MainWindow._open_settings` / `_on_settings_saved` — the dialog the update
  checkbox slots into.
- `cryptography 49.0.0` is present (transitive via pdfminer.six→pikepdf) and exposes
  `cryptography.hazmat.primitives.asymmetric.ed25519`. It is **not** in
  `_BANNED_NETWORK`, so importing it directly trips no network test; it is promoted
  to an explicit runtime dep (D1).

## Design decisions

- **D1 — Ed25519 via the already-bundled `cryptography` library; no minisign tool,
  no new runtime dependency.** The integrity gate is a detached **Ed25519**
  signature over the AppImage bytes. The app verifies it with
  `Ed25519PublicKey.from_public_bytes(...).verify(sig, data)` from `cryptography`
  (already in the bundle) — so **no external tool at runtime** and **no new install
  weight**. `cryptography` is promoted from transitive to an explicit
  `pyproject` runtime dep (we now import it directly; `dependencies.md`). The `.sig`
  is the **raw 64-byte** Ed25519 signature (binary), our own minimal convention —
  both the signer and verifier are ours, so the minisign container format
  (base64 + trusted-comment lines) would be parsing for no benefit. *Rejected:*
  shelling out to a `minisign`/`gpg` binary (can't assume it exists on the user's
  box or inside the bundle); a bare SHA-256 checksum (protects only against
  corruption, not a tampered release — wrong gate for a security app).

- **D2 — Simple full-file download + replace, not zsync/AppImageUpdate delta.** For
  a small, infrequently-updated app, downloading the whole AppImage and swapping it
  is dead simple and robust; delta updates (`zsync`) add a dependency and moving
  parts for a bandwidth saving that does not matter here (YAGNI). Surfaced, chosen
  deliberately.

- **D3 — Check once, on launch — no background polling.** One bounded outbound call
  per launch, only when opted in. No timer-driven polling (that multiplies the
  egress surface and the no-network posture). Predictable and auditable.

- **D4 — The opt-in flag + skipped-version live in the plaintext `window.ini`, NOT
  the vault.** The check must run **while locked** (before any unlock), so its
  settings cannot live in the encrypted vault (which needs the key to read). This is
  the deliberate mirror-image of FIBR-0055's auto-lock timeout (a *security control*,
  stored **in** the vault, read on unlock): the update prefs are **non-sensitive**
  (a boolean + a version string) and needed **pre-unlock**, exactly like the window
  geometry FIBR-0052 already keeps there. `UpdateService` owns the read/write via
  `QSettings(paths.window_settings_path(), IniFormat)`.

- **D5 — The toggle is surfaced in the existing Settings dialog (unlocked), value
  persisted to the plaintext INI.** No new pre-unlock settings UI. Default off; a
  user opts in once they reach **File → Settings…** (vault-dependent chrome). From
  the next launch the check reads the plaintext flag and runs even before unlock.
  `SettingsDialog` gains a checkbox; the shell persists it via
  `UpdateService.set_enabled(...)` on Save (the dialog holds no `UpdateService`
  network reference — it just reports the checkbox state), so the vault-scoped
  `AuthService.set_auto_lock_minutes` path (FIBR-0055) is untouched.

- **D6 — A platform seam: `Installer` protocol + `AppImageInstaller` +
  `detect_installer()`.** `Installer` (a `typing.Protocol`) is
  `can_self_update() -> bool` and `apply(new_file: Path) -> NoReturn`.
  `AppImageInstaller` implements the AppImage swap+relaunch; `detect_installer()`
  returns it only when `$APPIMAGE` is set (else `None` → INV-7). **This is the
  Windows plug-in point** — a future `WindowsInstaller` (download an installer /
  swap-on-exit helper) registers here with **no change** to `UpdateService` or the
  UI. Windows is *designed for*, not built (Out of scope).

- **D7 — The launch check runs on a `QThread` worker (mirroring `DeriveWorker`).**
  `ui/_update_worker.py::UpdateCheckWorker(QThread)` calls
  `UpdateService.check_for_update()` off the GUI thread and emits `found(UpdateInfo)`
  / `none()` / `failed(object)`; parented, `finished→deleteLater`, re-entrancy-safe.
  The shell arms it via `QTimer.singleShot(0, self._maybe_check_for_update)` from
  `__init__`, gated on `is_update_supported()` **and** `UpdateService.is_enabled()`.

- **D8 — Relaunch via `os.execv` after wiping the key.** `AppImageInstaller.apply`
  does: `chmod 0o755` the verified temp → `os.replace` over `$APPIMAGE` →
  `os.execv(appimage_path, [appimage_path])`. The shell calls the key-wipe
  (`AuthService.on_about_to_quit`) **before** delegating to `apply` (INV-6); `execv`
  never returns (typed `NoReturn`). *Rejected:* spawn-detached + `sys.exit` (leaves a
  brief window with two processes and an un-wiped parent); relying on `aboutToQuit`
  (execv bypasses it).

- **D9 — stdlib `urllib.request` for the API + download; no HTTP-client
  dependency.** The GitHub Releases API
  (`https://api.github.com/repos/milnet01/finbreak/releases/latest`) and the asset
  download use `urllib.request` with an explicit timeout and the INV-10 byte cap.
  Confined to `services/update_fetch.py` (D12). No `requests`/`httpx` — a new
  networked dependency would both bloat the bundle and widen the banned surface.

- **D10 — Amend `security-model.md` INV-8 to record the single consented egress.**
  INV-8 is rewritten from "no outbound call of any kind" to: "the shipped app makes
  **exactly one** kind of outbound call — an **opt-in, off-by-default** update check
  to the GitHub Releases API, confined to `services/update_fetch.py`; no other
  network access exists, and the check is never made without explicit user consent."
  The `pip-audit`/CI note is unchanged. This is a doc change tracked as a
  deliverable, not a silent drift.

- **D11 — Releases are published as full (non-prerelease) releases so
  `/releases/latest` returns them.** GitHub's `releases/latest` endpoint **excludes**
  prereleases; marking `v0.1.0` "pre-release" would make the check find nothing.
  They are early versions, not prereleases — published as normal releases. (An
  honest "preview" framing lives in the release notes, not the prerelease flag.)

- **D12 — All `urllib` use is isolated to `services/update_fetch.py`, the sole
  allowlisted networked module.** Keeping the network surface in one small,
  clearly-named file (whose only job is `fetch_latest_release()` +
  `download(...)`) makes the egress trivially auditable and lets the amended
  `test_INV8` allowlist name exactly one file for `urllib` only (INV-12).

## Deliverables

### Phase 1 — a real, signed Linux release

1. **`src/finbreak/__init__.py`** — `__version__ = "0.1.0"`.
2. **`scripts/gen-signing-key.py`** — a one-time helper the **user** runs: generates
   an Ed25519 keypair, writes the **private** key to a git-ignored path
   (`release/finbreak-signing.key`, printed to the user to safeguard), and prints the
   **public** key as the base64 constant to paste into `services/update_key.py`.
   Claude never sees or stores the private key. (INV-14)
3. **`src/finbreak/services/update_key.py`** — `RELEASE_PUBLIC_KEY_B64 = "<32-byte
   Ed25519 public key, base64>"` (filled in during Phase 1 from step 2) + a loader
   returning an `Ed25519PublicKey`. (INV-4/INV-14)
4. **`scripts/sign-release.py`** — signs an artifact with the private key (path via
   arg/env), writing `<artifact>.sig` (raw 64-byte Ed25519 signature). (INV-14)
5. **`scripts/build-release-appimage.sh`** (+ a `_build-release-in-container.sh`
   helper, or a `--release` flag on the existing build-smoke pair) — freezes the
   **real GUI** entry (not `--self-test`) into `finbreak-0.1.0-x86_64.AppImage`,
   reusing the FIBR-0003 PyInstaller + appimagetool container flow; then signs it
   (step 4). (INV-15)
6. **`.gitignore`** — ignore `release/finbreak-signing.key` (and any `*.key`
   private material). (INV-14)
7. **Publish `v0.1.0`** — `gh release create v0.1.0` with the AppImage + `.sig`
   attached, **not** marked prerelease (D11). (Done at implement time; may live in a
   thin `scripts/publish-release.sh` — the Linux slice of FIBR-0016.)

### Phase 2 — the in-app updater

8. **`src/finbreak/services/update_fetch.py`** *(the sole networked module)* —
   `fetch_latest_release(owner, repo, *, timeout) -> dict` (`urllib.request` GET of
   the Releases API, JSON-parsed) and `download(url, dest, *, max_bytes, timeout)`
   (streamed, INV-10 byte cap). The **only** file importing `urllib`. (INV-10/INV-12,
   D9)
9. **`src/finbreak/services/update.py`** — `UpdateService`:
   `is_enabled()`/`set_enabled(bool)` and `is_skipped(v)`/`skip_version(v)` (plaintext
   INI, D4); `check_for_update() -> UpdateInfo | None` (INV-1/2/3/8/11);
   `download_and_verify(info) -> Path` (Ed25519 verify, INV-4/10); plus `UpdateInfo`
   (`dataclass(frozen=True)`: `version: str`, `appimage_url`, `sig_url`,
   `notes_url`). Depends on an injected fetcher (default `update_fetch`), the
   plaintext-settings path, and `detect_installer()`.
10. **`src/finbreak/services/update_installer.py`** — the `Installer` `Protocol`,
    `AppImageInstaller` (`chmod`→`os.replace($APPIMAGE)`→`os.execv`, INV-5/6/8-swap),
    and `detect_installer() -> Installer | None` + `is_update_supported() -> bool`
    (INV-7, D6/D8).
11. **`src/finbreak/errors.py`** — `UpdateVerificationError` (and, if needed,
    `UpdateError`) alongside the existing vault errors. (INV-4)
12. **`src/finbreak/ui/update_dialog.py`** — `UpdateDialog(current, available,
    notes_url, parent)`: shows current vs available version + a "What's new" link
    (via `MainWindow._open_url`), three buttons — **Later** / **Skip this version** /
    **Update now** — plus the download-progress affordance; emits `later()` /

## Data model (plaintext INI; no vault schema change)

**No SQLCipher schema change, no migration.** Two new keys in the **plaintext**
`window.ini` (`paths.window_settings_path()`, FIBR-0052 INV-5) — non-sensitive,
readable before unlock:

| key | value | written by | read by |
|-----|-------|-----------|---------|
| `check_for_updates` | `"true"` / `"false"` | `UpdateService.set_enabled` (from Settings Save) | `UpdateService.is_enabled` (launch check) |
| `skipped_update_version` | a version string e.g. `"0.1.1"` | `UpdateService.skip_version` (Skip button) | `UpdateService.is_skipped` (check) |

Absent `check_for_updates` ⇒ **off** (INV-1). Absent `skipped_update_version` ⇒
nothing skipped.

## Out of scope (this cut)

- **Windows / macOS / Flatpak installers** — the `Installer` seam is built and
  Windows is *designed for* (D6), but only `AppImageInstaller` is implemented. On
  every non-AppImage package the feature is inert (INV-7). A `WindowsInstaller` is a
  follow-up (the user's "Windows too, later").
- **Delta / zsync updates** — full-file replace only (D2).
- **Background/periodic polling** — check on launch only (D3).
- **Auto-download without asking / silent updates** — always prompt; never force
  (roadmap bullet).
- **A full multi-platform release pipeline (P13/FIBR-0015/FIBR-0016)** — Phase 1
  builds only the Linux AppImage slice; the Windows `.exe`, macOS `.app`, and Flathub
  manifest remain FIBR-0015/0016.
- **In-app changelog rendering** — the prompt **links** to the GitHub release notes
  (via the browser); it does not fetch/parse/paint them.
- **Key rotation / multi-key trust** — a single baked-in public key; rotation is a
  future concern (would ship a new key in a new signed release).
