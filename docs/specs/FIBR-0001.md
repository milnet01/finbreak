# FIBR-0001 — P01 Bootstrap: lint/format/test/security harness

> `/cold-eyes` loop.
> **Phase:** P01 — Bootstrap. **Kind:** chore. **Source:** planned.
> **Roadmap:** [ROADMAP.md → FIBR-0001](../../ROADMAP.md).

**Theme:** stand up the build/dev plumbing and the **quality +
security gate** before any feature code exists, so the harness is
proven working and every later phase runs against it.

## Context

This is the first code phase. It ships **zero user-facing
features** on purpose. Its value is that the tooling that judges
all later work — linter, formatter, tests, and the
security scanners the user has made paramount — is known-good and
runs identically in CI and locally. Sibling P01 bullets
(`FIBR-0002` `.gitignore`/secret-leak guard, `FIBR-0003` bundling
smoke-test) are specced just-in-time when their turn comes.

## Invariants (testable)

- **INV-1 — One command, all gates, locally.** Running
  `scripts/ci-local.sh` from a clean checkout runs, in order,
  `ruff check`, `ruff format --check`, `bandit`, `pip-audit`,
  `gitleaks`, then `pytest`, and exits non-zero if **any** stage
  fails. Source: discovery.md "Local CI emulation".
- **INV-2 — CI runs the identical gate list.**
  `.github/workflows/ci.yml` runs the same stages in the same
  order as `scripts/ci-local.sh`. The gate list has **one source
  of truth** (the workflow invokes the script, or both read a
  shared list) so they cannot drift. Source: discovery.md.
- **INV-3 — Green on an empty project.** On the bootstrap commit
  (placeholder `src/` + empty test suite) every stage exits 0:
  `ruff` clean, `ruff format --check` clean, `bandit` 0 issues,
  `pip-audit` 0 known-vulnerable deps, `gitleaks` 0 leaks,
  `pytest` 0 collected/0 failed. Source: app-workflow P01
  definition.
- **INV-4 — Reproducible, pinned dev env.** `pyproject.toml`
  declares Python `>=3.12` and pins dev-tool versions
  (ruff, bandit, pip-audit, pytest, pytest-qt) so two machines
  resolve the same toolchain. Source: coding.md, global rule § 5.
- **INV-5 — Security stage is non-bypassable in the gate.** A
  deliberately-planted insecure pattern (e.g. a hardcoded secret
  fixture) makes `bandit`/`gitleaks` fail the gate; removing it
  makes the gate pass. This proves the security stage is wired,
  not stubbed. Source: security-model.md § 6, INV-6/INV-8.
- **INV-6 — Single-test ergonomics.** A named single test can be
  run in isolation (`pytest -k <name>` / `pytest <path>::<test>`)
  and the command is documented in `CLAUDE.md` "Build and test".
  Source: testing.md § 3.2.

## Deliverables

- `pyproject.toml` — project metadata, `>=3.12`, dev-dependency
  group, `[tool.ruff]`, `[tool.pytest.ini_options]` (markers:
  `features`, `integration`, `perf`), `[tool.bandit]`.
- `src/finbreak/__init__.py` — placeholder package so the linter
  and importer have a target.
- `tests/` — empty but collectable suite (a trivial
  `test_smoke.py` asserting the package imports).
- `scripts/ci-local.sh` — the one-command local gate (INV-1).
- `.github/workflows/ci.yml` — CI mirror (INV-2); `ubuntu-latest`
  for the lint/test/security gate (the build matrix arrives with
  packaging in P13).
- `CLAUDE.md` "Build and test" + "Module map" sections filled in.

## Out of scope

- Any application feature (that's P02 onward).
- The multi-OS build matrix and packaging (P13; smoke-test in
  `FIBR-0003`).
- Pre-commit hooks — the gate is the script + CI; a pre-commit
  hook can be added later if desired but is not required here.

## Exit criteria (Definition of Done)

1. `scripts/ci-local.sh` exits 0 on a clean checkout (INV-1,
   INV-3).
2. CI is green on the bootstrap commit and runs the same stages
   (INV-2).
3. The INV-5 planted-secret check demonstrably flips the gate
   red, then green when removed.
4. `/audit` and `/indie-review` return zero actionable findings
   on the same closing pass.
5. User approves closing P01's first item.

## Dependencies

None — this is the root of the build DAG.
